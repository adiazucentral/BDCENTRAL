Ruta de la muestra
Este modulo nos permite hacer el ingreso de una muestra en los diferentes destinos configurados para una  ruta del laboratorio.                    
Para que este modulo esta habilitado, el laboratorio tendra que manejar una trazabilidad completa.

A continuación, se muestra una ventana típica con sus partes:

 

  1. Destino donde se verifica la muestra de la orden ingresada (Inicio, proceso y final).
      Destino inicial
      Destino proceso externo
      Destino proceso interno
      Destino de control
      Destino final
  2. Caja de texto para  escanear el codigo de barras de una muestra o  ingresar el numero de la orden, seguido del separador de la muestra (configuración) y el código de la misma.
  3. Botonera para realizar diferentes acciones con la orden y/o muestra (cada bóton se explica con detalle mas adelante).  
  4. Número total de muestras ingresadas por todos los usuarios del LIS el día actual.
  5. Número total de muestras verificadas por todos los usuarios del LIS el día actual.
  6. Grafico de barras para comparar la cantidad de muestras en sus diferentes estados.  

Nota: la actualizacion de los items 2,3,4 se realizara segun la configuración del sistema. (Actualizacion widget (min) de la opción de trazabilidad).

Al ingresar el número de la orden sin una muestra, el sistema mostrará la siguiente información:

  1. Sección informativa sobre los datos del paciente relacionado a la orden
  2. Listado de muestras que se encuentran relacionadas a la orden
  3. Código de la muestra relacionada a la orden
  4. Imagen del tubo relacionado a la muestra, si esta no esta configurada el sistema cargara una imagen por defecto.
  5. Porcentaje de avance de la ruta configurada para la muestra, si esta no se encuentra configurada, el sistema mostrara en esta sección un mensaje informativo. 
  6. Información general de la muestra.
  7. Pruebas relacionadas a la muestra que le han sido asignadas a la orden
  8. Información del manejo de la muestra (Configuración).

Cada muestra podra tener diferentes estados:

Muestra verificada en diferentes destinos.

Muestra que lleva en el laboratorio un tiempo prolongado por lo tanto la calidad de la muestra puede afectar los analisis que se le realicen, este tiempo se configura por cada muestra al momento de la creación 

Muestra rechazada.

Muestra con retoma.

Cuando se ingrese una orden valida con una muestra que  no haya sido verificada con anterioridad y este relacionada a dicha orden, el sistema mostrara la siguiente información

  1. Ruta configurada para la muestra ingresada. cuando esta sea modificada en configuración y haya muestras verificadas con la ruta anterior, esta no será modificada.
  2. Fecha y hora en la que se verificó la muestra, cuando la muestra no ha sido verificada el texto que se mostrara será 'Pendiente'.
  3. Código y color configurado para el destino.
  4. Nombre configurado para el destino.
  5. Acción realizada y usuario que la realizo.
  6. Cantidad de pruebas asignadas a la orden que se procesan en ese destino, al dar click en el numero se mostrará el listado de pruebas con la siguiente información:

 

El botón de código de barras solo se habilitará cuando se haya ingresado una orden valida, al dar click en el se mostrara la siguiente información:

  1. Listado de muestras que corresponden a la orden, las marcadas con el check son las que van a ser mandadas a la impresora de etiquetas.
  2. Cantidad de etiquetas que van a ser mandadas por muestra, el valor por defecto es el que se haya configurado para cada muestra.
  3. Check para imprimir una etiqueta adicional en el listado de muestras.
  4. Botón para mandar las etiquetas a la impresora de códigos de barras

Nota: Para que la impresión de etiquetas sea funcional y el sistema reconozca la impresora se debe tener instalado el cliente de impresión en la maquina donde se desee imprimir.

El botón de rechazo solo se habilitará cuando se ingrese una muestra que haya sido verificada con anterioridad y no este en estado rechazado , esta ventana solicitará el motivo y un comentario por el cual desea realizar la acción .

El botón de retoma solo se habilitará cuando se ingrese una muestra que haya sido verificada con anterioridad y no este rechazada, esta ventana solicitará el motivo, el comentario por el cual desea realizar la acción y la seleccion de las pruebas que desea retomar.

El botón de demograficos solo se habilitará cuando haya ingresado una orden valida, este botón mostrará a detalle los datos del paciente, de la orden y la entrevista.
