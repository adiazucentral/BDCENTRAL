Visor de sucesos

El visor de sucesos es la herramienta que provee la aplicación para consultar los diferentes errores que se pueden generar en el sistema con información detallada que facilite las tareas de diagnóstico al grupo de soporte de la compañía.

A continuación, se muestra una ventana típica con sus partes: 


[Imagen de referencia - Visor de sucesos.]

1. Ordenamiento. 
Usa el control de ordenamiento para ordenar de menor a mayor o viceversa según el campo que desees ordenar solo al hacer click sobre el.  

2. Fecha Inicial.
 Selecciona la fecha inicial en que se filtran los errores , No se podrán consultar errores que se hayan generado en un periodo superior a 30 días.

3. Fecha final.
 Selecciona fecha final en que se filtran los errores , No se podrán consultar errores que se hayan generado en un periodo superior a 30 días.

4. Buscador.
Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos del visor de sucesos. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

5. Total errores. 
Este control muestra la cantidad de errores visualizadas en la lista.

Las filas de la lista son seleccionables de manera que si se quiere ver información detallada del error solo necesita dar click con lo que se generaría una imagen similar a la siguiente:


[Imagen de referencia - Información detallada del error.]

1. Fecha. 
Muestra la fecha en que se genero el error.  

2. Usuario.
 Muestra el usuario que se encontraba conectado cuando se genero el error.

3. URL.
 Muestra la URL que se encontraba conectado cuando se genero el error.

4. Detalle del error.
 Muestra al detalle el error que se genero.

5. Boton PDF.
 Este botón muestra en un pdf los datos que se muestran en pantalla.

6. Boton email. 
Este boton permite enviar un pdf via email con los datos que se muestran en pantalla y se generaria una imagen similar a la siguiente:

 


[Imagen de referencia - Boton email.]

Esta información debe enviarse al grupo de servicio de CLTech o a su distribuidor para que se inicien las actividades de diagnóstico y posterior corrección del error.
