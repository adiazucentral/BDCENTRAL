Perfil

El maestro de perfiles y paquetes que agrupan las pruebas en el sistema  

A continuación, se muestra una ventana típica con sus partes:

1.     Cuadro de búsqueda.                                                                                          

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de perfiles. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

 

2.     Ordenamiento.

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente. El ícono que aparece en cada columna cambia según el orden.

 

3.     Cantidad.  

Este control muestra la cantidad de perfiles o paquetes visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce un nuevo perfil o paquete. 

 

4.     Usuario- fecha de última actualización.                                                                                                     

Aquí se muestra la fecha y el usuario que modificaron el perfil o paquete que ha sido  seleccionado. Por defecto no aparece visible, solo cuando se selecciona una  fila de la lista de paquetes o perfiles.

 

5.     Botón ayuda.

Aquí se despliega el manual de cada página.

 

6.      Código .

Identificador del perfil en toda la aplicación , el cual no se puede repetir. Es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 10 caracteres, Es un campo requerido.

 

7.     Nombre.

Nombre que se mostrara en todas la opciones de la aplicación, el cual no se puede repetir. Es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres, Es un campo requerido.

 

8.    Abreviatura .

Sigla para identificar el perfil en toda la aplicación, el cual no se puede repetir. Es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 10 caracteres, Es un campo requerido.

 

9.     Genero.

Define el género al cual se aplica el perfil, y puede ser hombre,mujer o ambos.

 

10.  Nivel.

Nivel de complejidad del perfil, se empleara para filtro en estadística con precios .

 

11.  Paquete.

Este campo indica si el registro es perfil o paquete . si es perfil, se cargan solo las pruebas en la lista de pruebas filtrado por el area, si es

paquete se desahabilita el area y cargaran todas las pruebas  y perfiles activos

  

12.  Area.

Este campo filtra el listado de pruebas cuando se encuentre seleccionado la opción perfil.

 

13.  Estado.

Este botón Indica si registro está activo o no, al modificar el estado siempre hay aparece un cuadro de dialogo que pide la confirmación.

17.  Botón nuevo.

Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear una nuevo perfil o paquete. Si dichas cajas de texto tienen datos, se limpian.Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar y Reporte. Este botón por defecto debe estar habilitado.

 

18.  Botón editar.

Este botón permite editar perfil o paquete seleccionado de la lista de datos y es habilitado cuando se selecciona un perfil o paquete de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Reporte.

 

19.  Botón guardar.

Este botón permite guardar un perfil o paquete ya sea creada o modificada. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo y Reporte.

 

20.  Botón cancelar.

Este botón cancela, limpia y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Reporte.

 

21.  Botón Reporte .

Este botón imprime los perfiles o paquetes que se encuentran en la lista de datos. Se muestra un reporte con las mismas columnas y los mismos registros filtrados.

 

22.  Lista de perfiles.

Es una lista de datos que contiene tres campos: Nombre, paquete y Estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro y las inactivas estarán seleccionadas con rosado.
