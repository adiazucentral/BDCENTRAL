Area

Un área hace referencia a la distribución interna del laboratorio y segmenta la especialidad relacionada con el procesamiento de la muestra, ejemplos de áreas son química, microbiología, hormonas, citología, patología, etc.

A continuación, se muestra una ventana típica con sus partes:

 


[Imagen 01 - Formulario de áreas]

1. Buscador.
Escribe una palabra o frase en el cuadro de búsqueda para buscar un elemento. Dado que la búsqueda se inicia en cuanto empiezas a escribir, si por ejemplo escribe "B", todas las áreas cuyo nombre empiece por la letra B se mostrarán en la lista de áreas.

2. Ordenamiento.
Usa el control de ordenamiento para ordenar de menor a mayor o viceversa según el campo que desees ordenar solo al hacer click  sobre él.  

3. Cantidad.
Este control muestra la cantidad de áreas visualizadas en la lista.

4. Fecha - Usuario de última actualización.
Aquí se despliega la fecha y el usuario que modificaron la área que ha sido seleccionada.

5. Botón ayuda.
Aquí despliega la explicación de cada uno de los controles

6. Código.
Identificador único de las áreas, este campo solo acepta caracteres numéricos, lo máximo son 8 caracteres es un dato que siempre es requerido.

7. Abreviatura.
Identifica el área y se utiliza principalmente para la impresión de etiquetas, listados, informes y tableros se aconseja que no sea de más de 3 caracteres, aunque acepta un máximo de 5 caracteres, este campo es requerido y alfanumérico.

8. Nombre.
Nombre del área, por regla de negocio sólo se permite guardarlo en mayúscula. Acepta un máximo de 16 caracteres es alfanumérico sin caracteres especiales y es requerido.

9. Tipo.
Este campo determina una relación directa entre el área y módulos de la aplicación, se trata de una lista fija donde se muestran los siguientes valores: Laboratorio, Microbiología, Patología y Citología con el propósito de que los exámenes asociados al área que se está creando se puedan utilizar en estos módulos del sistema. Acepta un máximo de 8 caracteres es requerido.

10. Validación parcial.
Esta opción es únicamente para el área de microbiología, en las situaciones donde los exámenes que tienen resultados registrados y validados se necesiten modificar no por error en la emisión del primer reporte, sino por un segundo reporte se debe tener en cuenta que para este caso el usuario puede ser diferente al que valido el examen, el siguiente ejemplo puede aclarar más el propósito de esta funcionalidad.

El resultado de un hemocultivo es negativo a las 24 Horas, y se debe realizar una validación parcial, para que el médico pueda ver los resultados generados hasta ese momento, sin embargo a las 48 Horas se debe emitir un segundo informe, para este caso el resultado es positivo entonces el software lo que permite es que se pueda des-validar el primer resultado emitido y adicionar el nuevo, sin quedar registrado como un cambio de resultado en la trazabilidad.

11. Color.
Esta opción permite a la aplicación seleccionar un color para identificar el área. Es requerido para todas las áreas.

12. Estado.
Indica si el área se encuentra activa o inactiva. Cuando se crea una nueva área este toma el valor de activo. Este campo es booleano y requerido.

13. Botón nuevo.
Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear una nueva área. Si dichas cajas de texto tienen datos, se limpian. Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar y Reporte. Este botón por defecto está habilitado.

14. Botón editar.
Este botón permite editar la área selecciona de la lista de datos y es habilitado cuando se selecciona una área de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Reporte.

15. Botón guardar.
Este botón permite guardar una área ya sea creada o modificada. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botón Nuevo y Reporte.

16. Botón cancelar.
Este botón cancela, limpia y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Reporte.

17. Botón  Reporte.
Este botón imprime las áreas que se encuentran en la lista de datos. Se muestra un reporte con las mismas columnas y los mismos registros filtrados.

18. Listado de áreas.
Es una lista de datos que contiene cuatro campos: Código, Nombre, Tipo y Estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro.
