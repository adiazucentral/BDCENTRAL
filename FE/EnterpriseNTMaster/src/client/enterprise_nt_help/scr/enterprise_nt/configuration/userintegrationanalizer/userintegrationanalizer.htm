Integracion - Analizadores

Define los usuarios analizadores y/o de integración que se utilizaran en toda la aplicación

A continuación, se muestra una ventana típica con sus partes:

 

 

1. Cuadro de búsqueda.

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de usuarios. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

 

2. Ordenamiento.

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente. El ícono que aparece en cada columna cambia según el orden.

 

3. Cantidad.

Este control muestra la cantidad de usuarios visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce un nuevo usuarios.

                                                                                                          

4. Usuario- fecha de última actualización.                                                                                                      

Aquí se muestra la fecha y el usuario que modificaron el usuario que ha sido  seleccionado. Por defecto no aparece visible, solo cuando se selecciona una  fila de la lista de destinos.

 

5. Botón ayuda.

Aquí se despliega el manual de cada página. 

 

6.  Foto.                                                                                                      

       Es un campo especial para cargar la foto del usuario que se va a crear o que se va a modificar, no es obligatorio.

 

7. Tipo de usuarios.

Identifica el tipo de usuario que se va a ser creado o modificado si es analizador o un usuario que se utilizara para integraciones con diferentes aplicaciones.

cuando el usuario es tipo de tipo integración se le agregan el combo destino que verifica, y el laboratorio, y de analizador las contraseña y la confirmacion de la contraseña.

 

8. Identificación.

Es un campo queidentifica al usuario que se está gestionando, es requerido.

 

9. Nombre

       Es un campo alfa-numérico que hace referencia al nombre o nombres del usuario del sistema y es requerido.

 

10. Usuario

       Es un campo que es elIdentificador del usuario que se emplea para acceder al sistema; es requerido y se valida si está repetido en otro usuario.

 

       11. Activación usuario.

       Es donde al usuario se le asigna una fecha de activación que no debe ser mayor a la fecha de expiración del mismo y determina desde cuando el usuario puede hacer uso del sistema.

 

       12.  Expiración usuario.

       Es donde al usuario se le asigna una fecha de caducidad o expiración que no debe ser menor que la fecha de activación del mismo y determina hasta cuando el usuario puede tener acceso

       al sistema. Este campo es opcional y para los casos donde no se diligencie significa que el usuario no expira.

 

       13.   Contraseña.

       Es un campo que hace referencia a la clave para permitir el acceso al sistema. Este campo utiliza la máscara de contraseña de manera que el valor no sea visible.

       Cuando se edita un usuario no se visualiza y si se llega a guardar vacío en modo de edición la contraseña no se actualiza. Cuando se crea el usuario este campo si pasa a ser requerido.

       Cuando el tipo de usuario es analizador este campo no se visualiza.

   14.  Confirmar.

    Es el campo donde se confirma la clave o contraseña del usuario que se está creando o actualizando. Acá se valida si la confirmación coincide o no con la contraseña previamente digitada, si no

   coincide, sale un mensaje indicando el suceso y la fecha de expiración de la contraseña queda en blanco, si las contraseñas coinciden, aparece la fecha de la expiración de la contraseña. Este

   campo utiliza la máscara de contraseña de manera que el valor no sea visible. Cuando se edita un usuario no se visualiza y si se llega a guardar vacío en modo de edición la confirmación de

   la contraseña no se actualiza. Cuando se crea el usuario este campo si pasa a ser requerido.

   Cuando el tipo de usuario es analizador este campo no se visualiza.

 

   15.  Expiración contraseña.

    Indica la fecha de caducidad o expiración de la contraseña del usuario. Esta fecha es calculada automáticamente con los días programados para su expiración. Este campo es solo informativo, no

   se diligencia y solo aparece cuando se confirma la contraseña. Cuando el tipo de usuario es analizador este campo no se visualiza.

 

   16. Estado

   Este botón Indica si registro está activo o no, al modificar el estado siempre hay aparece un cuadro de dialogo que pide la confirmación.

17. Botón Reporte

            Este botón imprime los usuarios que se encuentran en la lista de datos. Se muestra un reporte con las mismas columnas y los mismos registros filtrados.

 

18. Botón cancelar

Este botón cancela, limpia y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Imprimir.

 

19. Botón guardar

Este botón permite guardar un destino  ya sea creada o modificada. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo e Imprimir.

 

20. Botón editar

Este botón permite editar los destinos seleccionados de la lista de datos y es habilitado cuando se selecciona un usuarios de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Imprimir.

 

21. Botón nuevo

Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear una nuevo usuario. Si dichas cajas de texto tienen datos, se limpian. Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar e Imprimir. Este botón por defecto debe estar habilitado.

 

22. Lista de usuarios

Es una lista de datos, dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro y  las inactivas estarán resaltadas con rosado.
