Demográfico adicional - consulta web

El maestro demográfico adicional de consulta web, permite configurar a los item del demográfico configurado el usuario y la contraseña para loquearse en consulta web  

A continuación, se muestra una ventana típica con sus partes:

 

1. Ordenamiento.

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente. El ícono que aparece en cada columna cambia según el orden.  

      2. Cuadro de búsqueda de los ítems  demográfico.                                                                                         

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

 

3. Demográfico.

Indica el demográfico al cual se aplicara el filtro, este demográfico es configurable en dependencia demográficos. 

4. Cantidad.

Este control muestra la cantidad de datos visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda.

 

5. Botón ayuda.

Aquí se despliega el manual de cada página.

 

6.  Usuario.

Es un campo que es el Identificador del usuario que se emplea para acceder al sistema; es requerido y se valida si está repetido en otro usuario.

 

7.  Contraseña.

Es un campo que hace referencia a la clave para permitir el acceso al sistema. Este campo utiliza la máscara de contraseña de manera que el valor no sea visible. Cuando se edita un usuario no se visualiza y si se llega a guardar vacío en modo de edición la contraseña no se actualiza. Cuando se crea el usuario este campo si pasa a ser requerido.

 

8. Confirmar.

Es el campo donde se confirma la clave o contraseña del usuario que se está creando o actualizando. Acá se valida si la confirmación coincide o no con la contraseña previamente digitada, si no coincide, sale un mensaje indicando el suceso y la fecha de expiración de la contraseña queda en blanco, si las contraseñas coinciden, aparece la fecha de la expiración de la contraseña. Este campo utiliza la máscara de contraseña de manera que el valor no sea visible. Cuando se edita un usuario no se visualiza y si se llega a guardar vacío en modo de edición la confirmación de contraseña no se actualiza. Cuando se crea el usuario este campo si pasa a ser requerido.

 

9. Expiración contraseña.

Indica la fecha de caducidad o expiración de la contraseña del usuario. Esta fecha es calculada automáticamente con los días programados para su expiración. Este campo es solo informativo, no se diligencia y solo aparece cuando se confirma la contraseña.      

 

10.Botón editar

Este botón permite editar los datos del item demografico seleccionados de la lista de datos y es habilitado cuando se selecciona un destino  de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar , se habilitan todos los controles del formulario. Se habilitan los botonesGuardar yCancelar.

 

11. Botón guardar.

Este botón permite guardar la relación entre prueba y el ítem demográfico seleccionado. Al pulsarlo se deshabilita el botón Cancelar y el propio botón, y se habilita el  botón  reporte.

 

12. Botón cancelar.

Este botón cancela y vuelve los datos como se encontraban guardados. Deshabilita el botón Guardar y habilita el botón Reporte

 

13. Lista de ítem demográfico.

Es una lista de datos que contiene los ítems demográficos asociados al demográfico que está configurado. Dicha lista puede ser ordenada y/o filtrada. La fila seleccionada siempre se mostrará re saltada en azul.
