Requisitos

El maestro de requisitos va asociado con los exámenes, y hace referencia a las condiciones que requiere tener el paciente para poder tomar un examen determinado.
A continuación, se muestra una ventana típica con sus partes:


1. Cuadro de texto de búsqueda.                                                                                           
Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de los requisitos. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

2. Ordenamiento.
Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente o por valor (falso o verdadero). El ícono que aparece en cada columna cambia según el orden.

3. Cantidad.
Este control muestra la cantidad de requisitos visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce un nuevo requisito.

4. Fecha de última actualización - Usuario.                                                                                                      
Aquí se muestra la fecha y el usuario que creó o modificó el requisito que ha sido  seleccionado. Por defecto no aparece visible, solo cuando se selecciona una fila de la lista de datos.

5. Botón ayuda.  
Aquí se despliega el manual de cada página.

6. Requisito.                                                                                                      
Es un campo alfa-numérico que identifica un requisito para hacer exámenes, acepta mayúsculas, minúsculas y texto enriquecido; es un dato que siempre será requerido y validado en caso de estar repetido.

7. Descripción.
Es un campo tipo texto multi-lineal, el cual almacena el detalle del requisito, que puede ser impreso en el informe de requisitos para la toma del examen.

8. Estado.
Indica si el requisito se encuentra activo o no. Al cambiar el estado muestra un cuadro de confirmación donde pregunta si se encuentra seguro de cambiar el estado o no. El control tiene forma de interruptor (switch).

9. Botón nuevo.
Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear un nuevo requisito. Si dichas cajas de texto tienen datos, se limpian.
Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar y Reporte . Este botón por defecto debe estar habilitado.

10. Botón editar.
Este botón permite editar la requisito selecciona de la lista de datos y es habilitado cuando se selecciona una requisito de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Reporte .

11. Botón guardar.
Este botón permite guardar una requisito ya sea creada o modificada. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo y Reporte .

12. Botón cancelar.
Este botón cancela, limpia y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Reporte .

13. Botón reporte.
Este botón abre una ventana donde se muestra el reporte de los requisitos que se encuentran en la lista de datos. Se muestra los mismos registros filtrados y las opciones para guardar ya sea en PDF o en Excel.

14. Listado de requisitos.
Es una lista de datos que contiene tres campos: Código, Descripción y Estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro. Las filas inactivas se motrarán resaltadas en rosado.
