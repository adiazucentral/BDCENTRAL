Integración - Analizador

Este maestro es el encargado de definir los usuarios para las integraciones y los analizadores.

A continuación, se muestra una ventana típica con sus partes:

1. Ordenamiento.

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente o por el estado (falso o verdadero). El ícono que aparece en cada columna cambia según el orden.

 

2. Cuadro de búsqueda.                                                                                           

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles. 

 

3. Cantidad.

Este control muestra la cantidad de  datos visualizadas en la lista. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce un nuevo registro.         

 

4. Usuario- fecha de última actualización .                                                                                                     

Aquí se muestra la fecha y el usuario que modificaron el dato que ha sido  seleccionado. Por defecto no aparece visible, solo cuando se selecciona una fila de la lista de datos.       

 

5.Botón ayuda.                                                                                                     

Aquí se despliega el manual de cada página.     

                                                     

6. Especialidad

Nombre de la especialidad a definir el cual no se puede repetir. Es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres, Es un campo requerido.

 

       7. Tipo de usuario.

Campo de selección que indica el propósito del usuario, se pueden manejar los siguientes tipos:

        ·    Analizador

        ·    Integración HIS.

 

            8.     Identificación.                                                                                                      

      Es un campo que identifica al usuario que se está gestionando, puede ser el número de cédula u otro valor semejante.

      

            9.    Nombre.                                                                                                      

      Es un campo alfa-numérico que hace referencia al nombre o nombres del usuario del sistema. (Este campo será utilizado en reportes de resultados).

    

      10.   Usuario.

      Es un campo que es el Identificador del usuario que se emplea para acceder al sistema; es requerido y se valida si está repetido en otro usuario.

 

11.  Activación usuario.

Es donde al usuario se le asigna una fecha de activación que no debe ser mayor a la fecha de expiración del mismo y determina desde cuando el usuario puede hacer uso del sistema.

 

12.  Expiración usuario.

Es donde al usuario se le asigna una fecha de caducidad o expiración que no debe ser menor que la fecha de activación del mismo y determina hasta cuando el usuario puede tener acceso al sistema. Este campo es opcional y para los casos donde no se diligencie significa que el usuario no expira.

 

13.  Destino que verifica.

Destino en donde verifica el usuario tipo analizador.

 

14. Estado

Este botón Indica si registro está activo o no, al modificar el estado siempre hay aparece un cuadro de dialogo que pide la confirmación.

 

15. Botón nuevo

Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear un nuevo dato. Si dichas cajas de texto tienen datos, se limpian. Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar e Imprimir. Este botón por defecto debe estar habilitado.

 

16. Botón editar

Este botón permite editar el dato seleccionada de la lista de datos y es habilitado cuando se selecciona una dato de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Imprimir.

 

17. Botón guardar

Este botón permite guardar e l usuario ya sea creada o modificada. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botón Nuevo e Imprimir.

 

18. Botón cancelar

Este botón cancela, limpia y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Imprimir.

 

19. Botón reporte

Este botón imprime las usarios que se encuentran en la lista de datos. Se muestra un reporte con las mismas columnas y los mismos registros filtrados.

 

20. Lista de los usuario de integración y analizador

Es una lista de datos que contiene tres campos: usuario, tipo de usuario y Estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro y los usuarios inactivas estarán resaltadas con rosado.
