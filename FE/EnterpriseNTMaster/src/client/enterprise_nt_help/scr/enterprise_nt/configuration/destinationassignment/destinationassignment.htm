Asignación de destinos

Define la ruta que debe seguir cada una de las muestras en el laboratorio y las pruebas que se procesan en cada destino

A continuación, se muestra una ventana típica con sus partes:


[Imagen 01 - Formulario de asignación de destinos]

 

TIPOS DE DESTINO:

  Destino tipo inicial.

  Destino tipo proceso externo.

  Destino tipo proceso interno.

  Destino tipo control.

  Destino tipo final.

1. Buscador.
Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de destinos. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

2. Muestra.
Indica la muestra a la cual se le va a definir la ruta.

3. Tipo de orden. 
Indica si la ruta se aplicará a un tipo de orden específico o a todas.

4. Sede
Indica   la sede a la cual se le va a definir la ruta.

5. Cantidad.
Este control muestra la cantidad de áreas visualizadas en la lista.

6. Botón ayuda.
Aquí se despliega el manual de cada página. 

7. Destino inicial.
Destino inicial donde será procesada la muestra, la ruta debe contener mínimo un destino de este tipo para que sea válida.

8. Destino invalido.
Destino de tipo proceso interno o externo que no tiene pruebas asignadas, por lo tanto no es una ruta valida.

9. Ordenamiento.
Identifica el orden de los destinos por donde la muestra será procesada.

10. Cantidad de pruebas asignadas.
Número de pruebas asignadas a los destinos de tipo proceso interno o externo.

11. Destinos de proceso interno, proceso externo y control.
Listado de destinos de tipo interno, externo y control por donde la muestra será procesada, la ruta debe contener mínimo un destino de este tipo para que sea válida, los destinos de tipo proceso interno y externo deben contener mínimo una prueba asignada.

Para asignar pruebas a los destinos dar click sobre él, se mostrara una venta emergente como la siguiente:   

11.1 Listado de pruebas: Listado de pruebas para asignar al destino seleccionado.

11.2 Filtro por área: Lista de área existentes y activas en el sistema para filtrar el listado de pruebas.

11.3 Filtro de búsqueda de pruebas: Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de pruebas. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

11.4 Icono de bloqueo: Icono informativo para las prueba se encuentra asociada a otro destino de la misma ruta.

11.5 Asignación: Control para asignar la prueba al destino seleccionado.

11.6 Botón cancelar: Este botón deshace todos los cambios hechos y que no se hayan guardado con anterioridad.

11.7 Botón aceptar: Este botón asocia las pruebas seleccionadas al destino seleccionado.

12. Destinos finales.
Destinos finales donde la muestra terminara su ciclo, la ruta debe contener minino uno y máximo dos destino de este tipo para que sea válida.

13. Botón guardar.
Este botón permite guardar la ruta ya sea creada o modificada.

NOTA: Una ruta modificada tiene el comportamiento de una vigencia, se emplea para las nuevas muestras. Las rutas que están en proceso, se emplearan hasta que su ciclo finalice.

14. Botón cancelar.
Este botón cancela los cambios que se le hayan realizado a la ruta y no se hayan guardado.

15. Botón Reporte.
Este botón abre una ventana emergente que muestra el informe de la ruta, permite exportar en PDF, Excel o imprimir.

16. Listado de destinos.
Es una lista de datos que contiene los destinos existentes y activos, para ser asignados a la ruta seleccionada.
