Tipos de documento

Este maestro define los tipos de documento para la historia clínica del paciente.
Se emplea en ingreso, informes, registro de resultados, estadísticas, Rips, entre otros informes

A continuación, se muestra una ventana típica con sus partes:

  
 
1. Ordenamiento.
Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente. El ícono que aparece en cada columna cambia según el orden.

2. Cuadro de búsqueda.                                                                                           
Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de tipos de documento. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

3. Cantidad.
Este control muestra la cantidad de tipos de documento visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce un nuevo tipo de documento. 

4. Usuario- fecha de última actualización.                                                                                                      
Aquí se muestra la fecha y el usuario que modificaron el tipo de documento que ha sido  seleccionado. Por defecto no aparece visible, solo cuando se selecciona una  fila de la lista de tipos de documento.

5. Botón ayuda.
Aquí se despliega el manual de cada página. 

6. Abreviatura.
Abreviatura de los tipos de documento a definir. es un campo unico, alfa-numérico que acepta mayúsculas y minúsculas, con un máximo de 10 caracteres, Es un campo requerido.

6. Nombre . 
nombre  de los tipos de documento a definir. es un campo unico, alfa-numérico que acepta mayúsculas y minúsculas, con un máximo de 60 caracteres, Es un campo requerido.

8. Estado.
Este botón Indica si registro está activo o no, al modificar el estado siempre hay aparece un cuadro de dialogo que pide la confirmación.

9. Botón nuevo.
Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear un nuevo tipo de documento. Si dichas cajas de texto tienen datos, se limpian. Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar y reporte. Este botón por defecto debe estar habilitado.

10. Botón editar.
Este botón permite editar el tipo de doecumento seleccionado de la lista de datos y es habilitado cuando se selecciona un tipo de documento de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Reporte .

11. Botón guardar.
Este botón permite guardar un tipo de documento ya sea creado o modificado. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo y Reporte . 

12. Botón cancelar.
Este botón cancela, limpia y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Reporte.

13. Botón reporte.
Este botón abre una ventana emergente que muestra el informe de tipos de documento, permite exportar en PDF, EXCEl o imprimir los tipos de documento que se encuentran en la lista de datos con las mismas columnas y los mismos registros filtrados.

14. Lista de tipos de documento.
Es una lista de datos que contiene tres campos:Abreviatura, nombre y estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro y los tipos de documento inactivos estarán resaltadas con rosado.
