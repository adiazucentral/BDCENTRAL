Usuarios

El maestro de usuarios permite configurar la información de los usuarios del sistema, su rol, permisos, vigencia y sedes para especificar el acceso y manejo de la aplicación.

A continuación, se muestra una ventana típica con sus partes:

           

      1. Cuadro de texto de búsqueda.

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de los usuarios. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

 

      2.Ordenamiento.

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente o por valor (falso o verdadero). El ícono que aparece en cada columna cambia según el orden.
 

      3. Cantidad.

Este control muestra la cantidad de usuarios visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce un nuevo usuario.

 

      4.Fecha de última actualización - Usuario.                                                                                    

Aquí se muestra la fecha y el usuario que modificó el usuario seleccionado. Por defecto no aparece visible, solo cuando se selecciona una fila de la lista de usuarios.


      5. Botón ayuda.

Aquí se despliega el manual de cada página.

 

      6.    Foto.                                                                                                      

Es un campo especial para cargar la foto del usuario que se va a crear o que se va a modificar, no es obligatorio.

 

      7.     Identificación.                                                                                                      

Es un campo que identifica al usuario que se está gestionando, puede ser el número de cédula u otro valor semejante. No es requerido.

 

8.   Usuario.

Es un campo que es el Identificador del usuario que se emplea para acceder al sistema; es requerido y se valida si está repetido en otro usuario.

 

      9.    Nombre.                                                                                                      

Es un campo alfa-numérico que hace referencia al nombre o nombres del usuario del sistema. (Este campo será utilizado en reportes de resultados) y es requerido.

 

      10 .Apellido.

Es un campo alfa-numérico que hace referencia al apellido o apellidos del usuario del sistema. (Este campo será utilizado en reportes de resultados) y es requerido.

 

11.   Código de firma.

Es un campo texto que guarda un código de firma digital registrado, este campo es informativo y en los casos donde se registre se muestra en los informes de resultados donde aparecen los datos del usuario.

 

12.   Firma.

Es un espacio donde se muestra la Imagen de la firma del usuario que se utiliza en los informes de resultados con el proceso de validación. No es obligatorio, si está vacío no sale en los reportes.     

 

      13.  Correo.

Es donde al usuario se le define un e-mail o correo electrónico que sirve para la recuperación de la clave  del sistema. El campo valida el formato del correo para mejorar la calidad del dato.

 

14.   Estado.

Es un campo que permite activar o desactivar un usuario. Al cambiar el estado muestra un cuadro de confirmación donde pregunta si se encuentra seguro de cambiar el estado o no. El usuario principal no se puede desactivar.

 

15.   Permisos generales.

Es un panel donde se despliega información adicional del usuario en gestión. Se visualiza la siguiente imagen:

 

       15.1    Permiso para ingresar a tableros.

  Indica si el usuario tiene permisos para ingresar a tableros .

  15.2   Descuento %.

      Campo numérico que indica el descuento máximo que el usuario puede asignar en el módulo de caja de ingreso. El valor mínimo es 0 y mayor es 100.   

       15.3   Adicionar y quitar exámenes en registro de resultados.

  Indica si el usuario puede crear o no exámenes o perfiles en el módulo de registro de resultados.

       15.4   Confidencial.

  Indica si el usuario tiene permiso de consultar, ingresar y validar resultados de exámenes confidenciales.

       15.5   Editar información de pacientes en ingreso de órdenes.

  Indica si el usuario puede o no modificar la información del paciente desde el ingreso de órdenes incluyendo sus datos demográficos. Sin embargo, si desde la configuración de los demográficos la modificación está inactiva, prevalecerá la restricción de dicha edición en la información de los demográficos del paciente en el ingreso de órdenes. Por eso al tratar de cambiar el estado se muestra un mensaje de advertencia.

       15.6     Impresión de informes.

Indica si el usuario puede imprimir informes o no en cualquiera de los siguientes  módulos:

·       Registro de resultados

·       Informes y consultas

·       Control de entrega de Informes

·       Consulta Web

  ·       Aplica a Vistas preliminares, informes previos e informes finales.

       15.7    Creación de demográficos en ingreso de órdenes.

Esta configuración permite al usuario crear ítems demográficos en el módulo de ingreso de órdenes. Sin embargo desde el maestro de demográficos si la configuración de la creación de ítems demográficos está deshabilita este permiso también persistirá deshabilitado, es decir, el usuario no podrá activar dicho permiso porque depende de la configuración de los demográficos.

       15.8    Segunda validación.

Esta configuración indica si el usuario puede realizar una segunda validación de un resultado.

       15.9    Imprimir resultados en ingreso de órdenes.

Permite al usuario hacer una impresión de los resultados preliminares existentes desde el módulo de ingreso de órdenes al consultar una orden existente.

       15.10    Deshacer validación.

Indica si el usuario puede o no quitar la validación de un examen que ya se haya validado o impreso.

       15.11    Tipo de orden por defecto.

Muestra el tipo de orden que le aparece al usuario por defecto para facilitar las opciones de digitación (ingreso, listados informes).

Este valor proviene de la configuración del maestro de tipos de órdenes.

 

16.   Asignación de roles.

Se despliega el panel y se visualiza la lista de roles activos. En esta configuración se muestra los roles asociados al usuario, no se puede hacer el guardado sin que esté asociado por lo menos a un rol.

 

16.1 Rol.

Es la columna donde aparecen los nombres de los roles activos en el sistema.

16.2  Adicionar.

Es la columna donde al usuario se le asignan o se le adicionan roles activos en el sistema.

     

17.  Seguridad .

Se despliega el panel y se visualizan la contraseña, la confirmación, la fecha de expiración de las contraseña y la fechas de activación y expiración del usuario. 

 

        

17.1  Activación usuario.

Es donde al usuario se le asigna una fecha de activación que no debe ser mayor a la fecha de expiración del mismo y determina desde cuando el usuario puede hacer uso del sistema.

17.2  Expiración usuario.

Es donde al usuario se le asigna una fecha de caducidad o expiración que no debe ser menor que la fecha de activación del mismo y determina hasta cuando el usuario puede tener acceso al sistema. Este campo es opcional y para los casos donde no se diligencie significa que el usuario no expira.

17.3.  Contraseña.

Es un campo que hace referencia a la clave para permitir el acceso al sistema. Este campo utiliza la máscara de contraseña de manera que el valor no sea visible. Cuando se edita un usuario no se visualiza y si se llega a guardar vacío en modo de edición la contraseña no se actualiza. Cuando se crea el usuario este campo si pasa a ser requerido.

17.4.  Confirmar.

Es el campo donde se confirma la clave o contraseña del usuario que se está creando o actualizando. Acá se valida si la confirmación coincide o no con la contraseña previamente digitada, si no coincide, sale un mensaje indicando el suceso y la fecha de expiración de la contraseña queda en blanco, si las contraseñas coinciden, aparece la fecha de la expiración de la contraseña. Este campo utiliza la máscara de contraseña de manera que el valor no sea visible. Cuando se edita un usuario no se visualiza y si se llega a guardar vacío en modo de edición la confirmación de contraseña no se actualiza. Cuando se crea el usuario este campo si pasa a ser requerido.

17.5  Expiración contraseña.

Indica la fecha de caducidad o expiración de la contraseña del usuario. Esta fecha es calculada automáticamente con los días programados para su expiración. Este campo es solo informativo, no se diligencia y solo aparece cuando se confirma la contraseña.

     

       18.    Asignación de áreas.

Se despliega el panel y se visualiza la lista de áreas activas. En esta configuración se determina lo que el usuario puede hacer en un área específica del laboratorio, la aplicación configura el permiso para registrar resultados y validarlos. Cuando se restringen estos permisos, el usuario sólo podrá consultar los resultados que se hayan generado. Es requerido activar al menos un área para el registro de resultados.

18.1  Todos.

Es el encabezado para seleccionar todas las áreas en registro de resultados y/o validarlas.

18.2  Área.

Es la columna donde aparecen los nombres de las áreas activas en el sistema.

18.3  Registro de resultados.

Es la columna donde se seleccionan las áreas que aparecerán para registrarle resultados.

18.4  Validar.

Es la columna donde se seleccionan las áreas para validar resultados. Solo se activan si el área correspondiente tiene activo un registro de resultado.

 

      

       19.    Asignación de sedes.

Se despliega el panel y se visualiza la lista de sedes activas. En esta configuración se permite asignar las sedes en las que el usuario puede realizar gestión de información, la aplicación debe permitir configurar si se permite el acceso y la impresión por bloque. Es requerido activar al menos una sede para el registro de resultados.

19.1   Sede.

  Es la columna donde aparecen los nombres de las sedes activas en el sistema.

19.2    Acceso.

  Es la columna donde se seleccionan las sedes a las cuales el usuario tendrá acceso.

19.3   Imprimir por lote.

  Es la columna que se refiere a la impresión por lote, se activa siempre y cuando se haya activado un acceso de sede.

 

       20.    Botón nuevo.

Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear un nuevo usuario. Si dichas cajas de texto tienen datos, se limpian.

Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar e Imprimir. Este botón por defecto debe estar habilitado.

 

       21.    Botón editar.

Este botón permite editar el usuario seleccionado de la lista de datos y es habilitado cuando se selecciona de la lista. Se cargan los datos en las cajas de texto y al pulsar el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Imprimir.

 

       22.    Botón guardar.

Este botón permite guardar un usuario ya sea creado o modificado. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo e Imprimir. 

 

       23.    Botón cancelar.

Este botón cancela, devuelve y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botones Imprimir y Nuevo.

 

       24.    Botón reporte

Este botón abre una ventana donde se muestra el reporte de los usuarios que se encuentran en la lista de datos. Se muestra los mismos registros filtrados y las opciones para guardar ya sea en PDF o en Excel.

 

       25.    Listado de usuarios.

       Es una lista de datos que contiene 4 campos: Usuario, Nombre, Apellido y Estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro. Las filas inactivas se mostraran resaltadas en rosado. El registro del usuario del sistema se mostrará resaltado con un ícono de candado, éste no se puede editar.
