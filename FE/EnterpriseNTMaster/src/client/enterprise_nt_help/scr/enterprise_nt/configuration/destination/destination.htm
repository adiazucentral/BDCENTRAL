Destino

Define los destinos por donde  pasan las muestra en un laboratorio

A continuación, se muestra una ventana típica con sus partes:

 

 

1. Cuadro de búsqueda.

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de destinos. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

 

2. Ordenamiento.

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente. El ícono que aparece en cada columna cambia según el orden.

 

3. Cantidad.

Este control muestra la cantidad de destinos  visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce un nuevo destino.

                                                                                                          

4. Usuario- fecha de última actualización.                                                                                                      

Aquí se muestra la fecha y el usuario que modificaron el destino  que ha sido  seleccionado. Por defecto no aparece visible, solo cuando se selecciona una  fila de la lista de destinos.

 

5. Botón ayuda.

Aquí se despliega el manual de cada página. 

 

6. Código.

Código que identifica el destino,  el cual no se puede repetir Es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 10 caracteres, Es un campo requerido.

 

7. Destino.

Nombre del destino,el cual no se puede repetir Es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres, Es un campo requerido.

 

8. Tipo

Indica a que proceso aplica el destino, y se selecciona de la siguiente lista.

     *  Inicial

     *  Proceso Interno

     *  Proceso Externo

     *  Final

     *  Control 

 

9. Descripción

Campo de texto creado para ingresar una breve explicación sobre la funcionalidad del destino. Es un campo alfa-numérico que acepta mayúsculas y minúsculas. 

 

10. Color

Definición de un color único que distinga el destino visualmente, se emplea en verificación completa de muestras.

 

11. Estado

Este botón Indica si registro está activo o no, al modificar el estado siempre hay aparece un cuadro de dialogo que pide la confirmación.

 

12. Botón nuevo

Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear una nuevo destino. Si dichas cajas de texto tienen datos, se limpian. Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar e Imprimir. Este botón por defecto debe estar habilitado.

 

13. Botón editar

Este botón permite editar los destinos seleccionados de la lista de datos y es habilitado cuando se selecciona un destino  de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Imprimir.

 

14. Botón guardar

Este botón permite guardar un destino  ya sea creada o modificada. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo e Imprimir.

 

15. Botón cancelar

Este botón cancela, limpia y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Imprimir.

 

16. Botón Reporte

Este botón imprime los destinos que se encuentran en la lista de datos. Se muestra un reporte con las mismas columnas y los mismos registros filtrados.

 

17. Lista de destinos

Es una lista de datos que contiene cuatro campos: Código, Destinó, Tipo y Estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro y  las inactivas estarán resaltadas con rosado.
