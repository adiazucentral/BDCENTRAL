Pruebas

El maestro de pruebas es el formulario encargado de gestionar los exámenes de laboratorio que serán ulitizados a lo largo de todo el proyecto. 

A continuación, se muestra una ventana típica con sus partes:

1.     Filtro de áreas.                                                                                          

Se selecciona un área del sistema para visualizar una lista de pruebas correspondiente.

 

2.     Cuadro de búsqueda.                                                                                          

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de las prubas. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

 

3.     Ordenamiento.

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente. El ícono que aparece en cada columna cambia según el orden.

 

4.     Cantidad.  

Este control muestra la cantidad de áreas que están en el cuadro desplegable y las pruebas visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce una nueva prueba. 

 

5.     Usuario- fecha de última actualización.                                                                                                     

Aquí se muestra la fecha y el usuario que modificaron la prueba que ha sido  seleccionada. Por defecto no aparece visible, solo cuando se selecciona una  fila de la lista de pruebas.

 

6.     Botón ayuda.

Aquí se despliega el manual de cada página.

 

7.      Código .

Identificador de la prueba en toda la aplicación el cual no se puede repetir. Es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 10 caracteres, Es un campo requerido.

 

8.    Abreviatura .

Sigla para identificar la prueba en toda la aplicación el cual no se puede repetir. Es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 10 caracteres, Es un campo requerido. No se pueden digitar las palabras reservadas GENDER, RACE, SIZE ni WEIGHT.

 

9.    Muestra .

Es un cuadro desplegable que contiene la lista de muestras disponibles para asignarle a una prueba . Este campo es requerido.

 

10.    Nombre.

Es como se lllama la prueba que se mostrara en la mayoría de las opciones y reportes de la aplicación, el cual no se puede repetir. Es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres, Es un campo requerido.

 

11.    Nivel .

Es un cuadro desplegable que contiene la lista de niveles de complejidad disponibles para asignarle a una prueba. Se empleara para filtro en estadística con precios.

 

12.    Genero.

Define el género al cual se aplica la prueba, y puede ser hombre, mujer o ambos.

 

13.  Edad.

Es un campo que indica la unidad en la que se medirá la edad; posee dos opciones: Años o Días. Si se selecciona Años, el rango máximo sería de 200, si se selecciona Días, el rango máximo sería de 365.

 

14.  Rango inicial (Rango en años - Rango en días).

Aparecen las dos cajas de texto que solo admiten valores numéricos, donde el primer rango se valida si es menor o mayor que el segundo rango;

dependiendo de la opción de la edad, el valor máximo del rango cambia.

 

15.  Estado.

Este botón Indica si registro está activo o no , al modificar el estado siempre aparece un cuadro de dialogo que pide la confirmación.

 

16.  Caracteristicas generales.

      Es un panel donde se muestran temas de estadísticas y facturación.

      A continuación, se muestra el panel con sus respectivos campos:

 

 

16.1 Participa en facturación. Indica si la prueba se tiene en cuenta o no para temas de facturación, caja, tarifa y vigencia.

16.2 Participa en estadísticas. Indica si la prueba se tiene en cuenta o no para los reportes y datos estadísticos de la aplicación. 

16.3 Multipicar por. Este campo depende de los valores de configuración de la prueba "Participa en facturación" y "Participa en estadísticas" de manera que si por lo menos uno de ellos está habilitado se puede incluir este valor indicando que para esos módulos la cantidad se ve afectada por este valor, por.ejemplo si se configura para la prueba este valor en 2 significa que para los procesos de facturación, estadística o ambos la prueba cuenta por 2.

16.4 Titulo estadístico.  Nombre del exámen que aparecerá en los informes estadísiticos o de facturación.Por defecto es el mismo nombre del exámen y se activa cuando se marca las opciones de estadística, facturación. 

16.5 Estadísticas procesados. Indica si la prueba se tiene en cuenta o no para ser procesada para datos estadísticos de la aplicación.

16.6 Título de grupo. Campo texto que sirve de utilidad al momento de la impresión de resultado, dividiendo perfiles en segmentos para mejorar la presentación.      

16.7 Procesamiento por. Indica la manera como se obtiene el resultado teniendo en cuenta los siguientes criterios:

Manual: el resultado se genera sin la participación del LIS y para lo único que se utiliza es para registrar el resultado, por ejemplo un RH donde el bacteriólogo aplica el reactivo a la muestra y por observación determina el resultado que transcribe en el LIS.

Enterprise: El LIS participa en la generación del resultado sea manualmente para el ingreso de los parámetros, o resultados o automático donde la información puede venir de una interfaz, bajo cualquiera de los dos casos la herramienta es la encargada de generar el resultado.

Externo: Este valor se utiliza cuando el procesamiento es por parte de laboratorios de referencia y se utiliza para identificar los exámenes que no son procesados por el laboratorio. Este campo es requerido para el procesamiento de resultados por analizador.

16.8 Confidencial. Indica si el examen requiere trato confidencial y sólo se puede registrar, validar e imprimir con permisos especiales. Los resultados de estos exámenes no se muestran en las consultas del LIS o web a menos que se espcifique en los requisitos .

16.9 Eliminar de perfil. Indica si el examen se puede eliminar de un perfil. Para los casos de creación del examen esta opción está deshabilitada y para los casos de edición sólo se habilita si el examen pertenece a un perfil. Este campo se emplea en registro de resultados.

16.10 Imprimir informe. Lista que indica si el examen se imprime, no se imprime o se imprime cuando se modifica (Impresión por modificación).

16.11 Ver en consultas. Indica si la prueba se puede ver en las consultas ya sean en el LIS o en las consultas web.

16.12 Ver en ingreso de órdenes. Permite mostrar la prueba en la lista de ingreso de órdenes, con el fin de poder seleccionarla.

16.13 Actividades básicas. Este campo ss para saber si la prueba se tiene en cuenta en fórmulas del reporte de alerta temprana. 

16.14 Almacenamiento especial. Indica que las muestras asociadas al examen requieren ser almacenadas y no descartadas (ej: HIV con resultado Positivos), de manera que se evite sean desechadas por accidente.

16.15 Consentimiento informado. Indica si a la prueba se aplicara el consentimiento informado.

          16.16 Alicuota. Indica si a la prueba se comporta como una alicuota.

 

 

17.   Procesamiento.

Es un panel donde se encuentran los datos de procesamiento por día de las pruebas. A continuación se muestra el panel con sus respectivos

campos: 

 

17.1  Días máximos para modificar después de validado.  

Hace referencia al número de días en que el usuario puede modificar el valor del resultado de la prueba, después de que esta se ha validado.

Si se deja en blanco no se validan los días.

17.2  Días de entrega.

Este valor indica en cuantos días se entrega un resultado desde la creación de la orden y teniendo en cuenta los días de procesamiento.

Este dato se emplea en el talón de reclamación y se calcula a partir de la fecha de creación de la orden y los días de procesamiento para que

el paciente sepa cuando se  entregarían sus resultados.   

17.3  Días máximos para modificar después de impreso.

Hace referencia al número de días en que el usuario puede modificar un resultado después de impreso.

Si se deja en blanco no valida los días.  Si se deja 0 no permite modificar.

17.4.  Vigencia del resultado.

Campo donde se define la vigencia en días que tiene un resultado de una prueba para generar una alerta en el ingreso de ordenes. Los valores que posee dicho campo son: N/A (No aplica), 30 Días, 60 Días y 90 Días.

17.5  Días de procesamiento.

Indica los días en que se procesa la muestra en el laboratorio.

El valor almacenado en la base de datos es una representación de los días de la semana donde:

Lunes = 1, Martes = 2, Miércoles = 3, Jueves = 4, Viernes = 5,  Sábado = 6, Domingo = 7,

Lo que se guargaría es una cmbinación de acuerdo a la configuración que se haga del examen, por ejemplo si se guarda el valor “45” significa que se procesa solamente los jueves y  viernes. 

17.6  Requiere validación preliminar.

Indica si la prueba debe ser pre-validada antes de pasar a estado de validación en el registro de resultados. 

 

18.   Configuración de resultados.

Es el panel donde a la prueba se le configuran los items para el registro de resultados.

A continuación, se muestra el panel con sus partes:

 

     

18.1 Tipo de resultado.

Opcion que indica si el examen seleccionado recibirá los resultados de tipo numérico o texto.

18.2  Unidad.

Indica las unidades de medida en que se muestra el resultado.

Es requerido si el tipo de prueba es numérica.

Se toma del maestro de unidades.

18.3 Técnica.

Indica la técnica empleada para el proceso de consecución del resultado.

Viene del maestro de Técnicas, y no es obligatorio para la creación del exámen o prueba.

Se emplea en la impresión de los resultados.

18.4 Volumen.

Valor numérico que indica la cantidad mínima de muestra necesaria para generar el resultado.

Es informativo y no es requerido.

Se utiliza en verificación de muestras.

18.5 Resultado automático.

Indica un valor que se ingresará como resultado al momento de crear el examen en una orden.

Debe ser concordante con el tipo der esultado.

18.6 Resultado en ingreso.

Si este valor se encuentra activo al crear un orden sera obligatorio darle un resultado a la prueba.

18.7 Imprimir gráfica de seguimiento. Indica si se permite imprimir gráfica de seguimiento para el examen, en los módulos de consultas o en registro de resultados. 

18.8  Factor de conversión.

Dato numérico, que multiplica el resultado del exámen en registro de resultados.

Se activa cuando el tipo de examen es numérico y sus decimales dependen de la cantidad definida en el campo decimales

18.9  Decimales.

Si el tipo de resultado es numérico, se habilita este campo, con el fin de indicar la cantidad de decimales que manejará el resultado cuando se procese.

18.10 Concurrencias.

Las concurrencias se utilizan en los casos donde se genera el resultado de una prueba o control del mismo; es necesario mas de un dato, se utilizan también para los casos donde algunas pruebas se incluyen en una fórmula para que con un cálculo matemático se genere el resultado.

 

 

18.11 Generar alarma por tendencia.

Configura la alarma que mostrara cuando la prueba se comporte con tendencia.

18.12  Fórmula.

Permite la configuración de formulas en el maestro de pruebas para el cálculo de resultado a partir de otros parámetros.

Éste representa la ecuación empleada para el cálculo del resultado del examen que se está configurando.

Es necesario asegurar que la fórmula es correcta sintácticamente de manera que se puedan hacer los cálculos matemáticos relacionados con e resultado.

En pantalla se muestra la abreviatura de los exámenes empleados sin embargo internamente se guarda el código del examen.

19. Comentarios.

En este panel se encientran 3 tipos de comentarios para la prueba. Son creados con texto enriquesido.

   

19.1 Comentario fijo.

Complementa al resultado o puede apoyar la interpretación del mismo y se muestra al momento de registrar un resultado.

Como es fijo se crea al momento de crear el examen, generalmente se definen valores de referencia no cuantificables en este campo u cualquier otra información relevante para el examen.

19.2 Comentario al imprimir la prueba.

Texto grande inherente al examen, que sólo se visualiza cuando se imprime el resultado (Informe final).

Solo se puede modificar en este formulario.

19.3 información general.

Texto largo que da información del examen.

Actualmente se emplea en registro de resultados de algunos laboratorios.

No es modificable desde otro formulario.

 

 20.  Requisitos.

 Basados en el maestro de requisitos, se selecciona de una lista los requisitos necesarios para la toma de este examen

 A continuación, se muestra una ventana típica con sus partes:

   

 

        

 

21.  Botón nuevo.

Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear una nuevo médico. Si dichas cajas de texto tienen datos, se limpian.Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar y Reporte. Este botón por defecto debe estar habilitado.

 

22.  Botón editar.

Este botón permite editar el médico seleccionado de la lista de datos y es habilitado cuando se selecciona un médico de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Reporte.

 

23.  Botón guardar.

Este botón permite guardar un médico ya sea creada o modificada. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo y Reporte.

 

24.  Botón cancelar.

Este botón cancela, limpia y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Reporte.

 

25.  Botón Reporte.

Este botón imprime los médicos que se encuentran en la lista de datos. Se muestra un reporte con las mismas columnas y los mismos registros filtrados.

 

26.  Lista de pruebas.

Es una lista de datos que contiene dos campos: Nombre, Estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro y las inactivas estarán seleccionadas con rosado.
