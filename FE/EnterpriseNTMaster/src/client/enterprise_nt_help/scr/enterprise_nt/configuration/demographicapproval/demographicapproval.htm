Homologación de demograficos

Permite asignar a cada uno de los demográficos un código central para la comunicación con otros sistemas. Debe estar agrupada por sistema central y por demográfico.

  A continuación, se muestra una ventana típica con sus partes:

      1. Cuadro de texto de búsqueda para demográficos.

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de los demográficos. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

 

2.Sistema central

Lista de seleccion para los sitemas centrales activos del sistema.

 

      3.  Cantidad sistemas centrales. 

Este control muestra la cantidad de registros cargados en la lista de sistemas cetrales.

 

4.Cantidad demográficos. 

Este control muestra la cantidad de registros visualizados en la lista de datos de demograficos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda.

 

5. Cuadro de texto de búsqueda para items demográficos.

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de los items demográficos. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

 

6.Cantidad items demográficos. 

Este control muestra la cantidad de registros visualizados en la lista de datos de los items demograficos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda.

 

      7.    Botón ayuda. 

Aquí se despliega el manual de cada página.

 

8.  Listado de items demográficos.

Es una lista de datos que contiene los campos de: nombre y codigo central. Dicha lista puede ser ordenada y/o filtrada por cada campo.

 

      9.  Botón exportar.

Este boton generar un archivo .csv para que pueda ser diligenciado con los respoectivos codigos de homologacion y ser importado en el sistema. 

 

      10.  Botón Importar.

Este botónabre una ventana donde se podra escoger el archivo en formato csv que se desea importar, el contenido del archivo debe contener tres columnas con los titulos "centralSystemid", "demographic/id", "demographicItem/id" y centralCode, seguido de los datos de los a ser importados, los textos no pueden contener el caracter (') puesto que el sistema no podria leer el archivo. 

 

Ejemplo:

 

 

 

Despues de que el sistema reciba el archivo, y todos los datos sean validos el sistema recargara la lista de los items demográficos, en caso contrario el sistema mostrara un listado de los datos que tienen inconsistencias, los campos no pueden estar vacios, los identificadores deben corresponder y los codigos no se pueden repetir.

 

importante: Se debe tener en cuenta la configuracion del separador de listas de la aplicacion, puesto que esta debe coincidir con el del archivo .csv que va a ser importado, en caso contrario el sistema no podra leer el archivo. si los textos contienen comas(,) o puntos(.) se recomienda configurar el separador como punto y coma(;).

      11.  Botón reporte.

Este botón abre una ventana donde se muestra una previsualizacion del reporte, en este se podra escoger en que formato se desea exportar el listado, este informe debe ser acorde al filtro y ordenamiento escogido en la tabla de datos del formulario inicial.

 

      12.  Listado de demograficos.

Es una lista de datos que contiene los el campo. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro. esta lista solo cargara los demograficos fijos del sistema y los demograficos que se hayan configurado como codificados.
