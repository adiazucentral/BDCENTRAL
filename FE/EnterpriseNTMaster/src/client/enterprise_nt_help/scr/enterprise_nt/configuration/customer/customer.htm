Clientes

Este formulario maestro se encarga de la configuración de los clientes del laboratorio, entiéndase cliente como otras empresas o personas que utilizan los servicios del laboratorio pero no como pacientes sino con fines comerciales.

A continuación, se muestra una ventana típica con sus partes:

1. 

 

     

 1.  Cuadro de búsqueda.  

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de clientes. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.    

 

2. Ordenamiento.  

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente o por valor (falso o verdadero). El ícono que aparece en cada columna cambia según el orden.

 

3. Cantidad.

Este control muestra la cantidad de clientes visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce un nuevo cliente.

4. Fecha de última actualización - Usuario.

Aquí se muestra la fecha y el usuario que modificaron el cliente que ha sido  seleccionado. Por defecto no aparece visible, solo cuando se selecciona una fila de la lista de clientes. 

 

5. Botón ayuda.

Aquí se despliega el manual de cada página.

6. Identificación.

Es un campo alfa-numérico que hace referencia al documento de identificación del cliente. No es un capo requerido, pero si se valida si está repetido.

7. Código EPS.

Es un campo alfa-numérico que hace referencia al código del cliente en el registro de salud. Este dato es requerido en el RIPS, es decir, el cliente que no se le diligencie este campo no aparecerá en el formulario del RIPS, sin embargo no es un dato requerido desde este maestro y se valida si es repetido.

8. Nombre.

Es un campo alfa-numérico que identifica un cliente en el sistema. Este se emplea para mostrarlo en el ingreso de órdenes y en cada una de las opciones que lo filtre. Es un dato que siempre será requerido y se valida si es repetido.

9. Nombre impreso.

Es un campo alfa-numérico que diligencia el nombre del cliente que aparecerá en los reportes y facturas. Puede ser diferente al campo Nombre. Es un dato que siempre será requerido, pero no se valida si es repetido. 

      10.  Dirección 1.

En este campo se digita la dirección del laboratorio. No es obligatorio y se puede repetir. 

      11.  Dirección 2.

En este campo se digita otra dirección opcional del laboratorio. 

      12.  Teléfono.

Es un campo alfa-numérico que cuenta con un formato de entrada configurado previamente por el sistema. Hace referencia al contacto de cliente. No es obligatorio pero si se valida su formato. 

      13.  Correo.

Es un campo alfa-numérico donde se almacena la dirección electrónica para el envío de los resultados. Requerido si se tiene habilitado el envío automático de resultados. El formato de e-mail es validado en este control. 

      14.  Fax.

Este campo es encargado de diligenciar el número de Fax del cliente. Requerido si se tiene habilitado el envío automático de resultados. 

      15.  Código postal.

Es un campo opcional. Es el número asignado a la zona para el envío de correspondencia. 

      16.  Usuario.

Es un campo donde se almacena el usuario para consultas web. Este campo no es requerido pero no se puede repetir. 

      17.  Contraseña.

Es un campo que sirve para consultas web y se visualiza con caracteres ocultos (*); es requerido si y solo si se está creando un nuevo cliente y si se diligencia el campo de usuario. 

      18.  Responsable.

En este campo se introduce el nombre de la persona que representa o es responsable del cliente gestionado. No es obligatorio y se puede repetir. 

      19.  Observaciones.

Es un campo opcional donde se comentan características del cliente o simplemente se completa información. 

      20.  Cliente particular.

Indica si el cliente si es particular o no. El control tiene forma de interruptor (switch). 

      21.  Estado.

Indica si el cliente se encuentra activo o no. Al cambiar el estado muestra un cuadro de confirmación donde pregunta si se encuentra seguro de cambiar el estado o no. El control tiene forma de interruptor (switch). 

 

22. Aplicar encriptación en el reporte.
Indica que todos los reportes que se impriman tendran contraseña.

      23.  Facturación general.

En este panel se encuentra información de facturación del cliente que se está gestionando. Se visualiza la siguiente imagen:

23.1.   Estado o departamento: Región geográfica donde está ubicado el cliente. Para el caso de Estados Unidos o países estatales hace referencia a los estados, en el campo puede ir el código o el nombre del departamento. En facturación este campo será requerido.

23.2.   Municipio o ciudad: Campo que hace referencia a la ciudad o municipio donde se encuentra el cliente, se puede digitar el código de la ciudad, abreviatura o el nombre. En facturación este campo será requerido.

23.3.   Colonia o barrio: Indica de manera más detallada la ubicación del cliente dentro de una ciudad o municipio, en algunos países se les llama colonia y en otros barrio. Este campo será requerido en facturación.

23.4.   Monto máximo Per Cápita: Valor máximo permitido para la facturación per cápita. En el control el valor por defecto es 0 y cuenta con formato de moneda.

23.5.   Monto actual: Valor calculado que muestra el valor facturado hasta el momento. Este valor depende de la facturación per cápita. En el control el valor por defecto es 0 y cuenta con formato de moneda.

23.6.   Monto alarma: Valor mínimo en el cual se debe empezar a mostrar una alarma en registro de resultado, con el fin de saber cuánto dinero queda en la facturación per cápita.

Depende del monto actual. En el control el valor por defecto es 0 y cuenta con formato de moneda.

23.7.   Descuento: Valor en porcentaje para el descuento que se aplica al cliente sobre la tarifa estipulada.

    

      24.  Facturación Estados Unidos.

      Este panel es visible solo cuando la llave de configuración de Facturación de USA está activa; aquí se encuentra información de facturación del cliente que se está gestionando. Al desplegarlo se visualiza la siguiente imagen

 

24.1.   Tipos de cuenta: Existen solo dos tipos de cuenta. Institucional (Valor por defecto) o Persona. Este campo será requerido en facturación de USA.

24.2.   Conectividad EMR.

Identifica si se envía el resultado a los clientes que tengan interfaz.

24.3.   Envío de resultados por FAX automático: Indica si al cliente se le enviarán los resultados automáticamente  por medio del Fax. Debe tener configurado número de fax.

24.4.   Envío de resultados por correo automático: Indica si se envía correo automático del resultado al cliente. Debe tener configurado el correo.

24.5.   Envío de resultados: Indica si se envía los informes completos de resultados o solo previos, ya sea por fax o por correo. El valor por defecto es Previos.

24.6.   Impresión de resultados: Indica si se crea una copia adicional para el cliente en forma impresa o solamente si con el envío es suficiente. 

      25.  Botón nuevo.

Este botón habilita los controles (excepto el interruptor de estado) para crear un nuevo clientes. Si dichos controles tienen información, se limpian.

Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar e Imprimir. Este botón por defecto debe estar habilitado. 

      26.  Botón editar.

Este botón permite editar el cliente seleccionado de la lista de datos y es habilitado cuando se selecciona un cliente de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Imprimir. 

      27.  Botón guardar.

Este botón permite guardar un cliente ya sea creado o modificado. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo e Imprimir.

      28.  Botón cancelar.

Este botón cancela, limpia y deshabilita los controles del formulario incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Imprimir.

      29.  Botón reporte.

Este botón abre una ventana donde se muestra el reporte de los clientes que se encuentran en la lista de datos. Se muestra el mismo número de registros filtrados, pero con el campo de Nombre Impreso y las opciones para guardar ya sea en PDF, Excel u otro formato.

      30.   Listado de clientes.

Es una lista de datos que contiene dos campos: Nombre y Estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro. La fila inactiva se mostrará en rosado.
