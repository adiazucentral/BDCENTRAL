Muestras

Las Muestras son los fluidos corporales en que se realiza el examen, y es un dato requerido para poder generar los códigos de barras para marcar los tubos y la trasmisión de resultados con analizadores.
A continuación, se muestra una ventana típica con sus partes:


 

1. Cuadro de búsqueda.                                                                                           
Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de muestras. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

2. Ordenamiento.
Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente. El ícono que aparece en cada columna cambia según el orden.

3. Cantidad.
Este control muestra la cantidad de muestras visualizadas en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce una nueva muestra.  
 
4. Usuario- fecha de última actualización.                                                                                                      
Aquí se muestra la fecha y el usuario que modificaron la muestra que ha sido  seleccionada. Por defecto no aparece visible, solo cuando se selecciona una  fila de la lista de muestras.

5. Botón ayuda. 
Aquí se despliega el manual de cada página.

6. Código.   
Identificador único de la muestra el cual no se puede repetir, De preferencia nemotécnico. Es un campo numérico; lo máximo son 3 caracteres y es un dato que siempre será requerido.

7. Nombre.
Nombre de la muestra, el cual no se puede repetir.Este dato se imprime en los códigos de barras. Es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres, Es un campo requerido.

8. Recipiente.
Campo de Lista que trae el nombre de los recipientes previamente creados y que se encuentren activos.

9. Cantidad de etiquetas.
Indica la cantidad de copias se imprimen en código de barras. Es un campo entero de máximo 2 caracteres.

10. Dias de almacenamiento.
Esta opción permite a la aplicación seleccionar los días de almacenamiento de la muestra, es una campo numérico y requerido.

11. Almacenamiento especial.
Esta opción permite guardar la muestra en la gradilla especial.

12. Tipo de laboratorio.
Este campo determina una relación directa entre la muestra y módulos de la aplicación, se trata de una lista fija donde se muestran los siguientes valores: Laboratorio, Microbiología, Patología y Citología.

13. Estado.
 Este campo Indica si registro está activo o no.

14. Información manejo de la muestra.
Campo informativo que se muestra en verificación de muestra completa, el cual indica el cuidado para el manejo de la muestra.Es un campo abierto de texto.

15. Verificación de la muestra.
 Pestaña donde se configura la muestra cuando se encuentra en verificación a continuación se muestra una ventana tipica con sus partes.

15.1. Requiere verificación.
Este campo Indica si la muestra se verifica en trazabilidad sencilla o completa de la muestra. Si no se verifica la muestra no se puede procesar por analizador o ingresar resultados Este campo se habilita si esta activa la opción de trazabilidad de muestra en el módulo de instalación.

15.2. Tiempo de calidad.
Este campo Indica el tiempo de calidad de la muestra desde que esta ingresa al laboratorio.

15.3. Alarma tiempo de calidad.
Este campo Indica el porcentaje en tiempo desde que la muestra ingreso al laboratorio hasta que el tiempo de calidad de la muestra se va a vencer, y mostrar una alarma para informar al usuario.

15.4. Temperatura inicial.
Este campo indica la temperatura mínima que puede tener la muestra, solo si se tiene habilitada la llave de temperatura.

15.5. Temperatura final.
Este campo Indica la temperatura máxima que puede tener la muestra, solo si se tiene habilitada la llave de temperatura.

15.6. Muestra con tapa.
Este campo Indica si la muestra contiene tapa o nop, este campo se utiliza cuando la muestra se procesa por analizador.

   

16. Botón Reporte .
Este botón imprime las muestras que se encuentran en la lista de datos. Se muestra un reporte con las mismas columnas y los mismos registros filtrados.

17. Botón cancelar.
Este botón cancela, limpia y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Reporte .

18. Botón guardar.
Este botón permite guardar una muestra ya sea creada o modificada. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo y Reporte .

19. Botón editar.
Este botón permite editar la muestra selecciona de la lista de datos y es habilitado cuando se selecciona una muestra de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Reporte .

20. Botón nuevo.
Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear una nueva muestra. Si dichas cajas de texto tienen datos, se limpian.
Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar y Reporte . Este botón por defecto debe estar habilitado.

21. Lista de muestras.
Es una lista de datos que contiene tres campos: Código, Nombre y Estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro y las muestras inactivas estarán seleccionadas con rosado.
