Tipos de pago

Permite la configuración de los diferentes tipos de pago que se pueden manejar en la aplicación este parámetro se utiliza para el módulo de facturación.

A continuación, se muestra una ventana típica con sus partes:

   


1. Cuadro de búsqueda.                                                                                           
Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de tipos de pago. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

2. Ordenamiento.
Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente. El ícono que aparece en cada columna cambia según el orden.

3. Cantidad.
Este control muestra la cantidad de tipos de pago visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce un nuevo tipo de pago. 

4. Usuario- fecha de última actualización.                                                                                                      
Aquí se muestra la fecha y el usuario que modificaron el tipo de pago que ha sido  seleccionado. Por defecto no aparece visible, solo cuando se selecciona una  fila de la lista de tipos de pago.

5. Botón ayuda.
Aquí se despliega el manual de cada página. 

6. Nombre . 
nombre  de los tipos de pagoa definir. es un campo unico, alfa-numérico que acepta mayúsculas y minúsculas, con un máximo de 60 caracteres, Es un campo requerido.

7. Banco . 
Indica si se activa en caja el banco para ser capturado.

8 Tarjeta de credito . 
Indica si se activa la captura del número de la tarjeta en caja.

9 Cuenta . 
Indica si permite la captura del número de cuenta a la que pertenece el banco, tarjeta o cheque en caja.

10. Suma . 
Indica si el valor ingresado en caja, suma al valor del pago en caja.

11. Estado.
 Este botón Indica si registro está activo o no, al modificar el estado, aparece un cuadro de dialogo que pide la confirmación.

12. Botón nuevo.
Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear un nuevo tipo de pago. Si dichas cajas de texto tienen datos, se limpian. Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar y reporte. Este botón por defecto debe estar habilitado.

13. Botón editar.
Este botón permite editar el tipo de pagp seleccionado de la lista de datos y es habilitado cuando se selecciona un tipo de documento de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Reporte .

14. Botón guardar.
 Este botón permite guardar un tipo de pago ya sea creado o modificado. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo y Reporte . 

15. Botón cancelar.
Este botón cancela, limpia y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Reporte.

16. Botón reporte.
Este botón abre una ventana emergente que muestra el informe de tipos de pago, permite exportar en PDF, EXCEl o imprimir los tipos de pago que se encuentran en la lista de datos con las mismas columnas y los mismos registros filtrados.

17. Lista de tipos de pago.
Es una lista de datos que contiene dos campos: Nombre y estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro y los tipos de documento inactivos estarán resaltadas con rosado, Se mostrara en gris el tipo de pago efectivo pusto que este solo sera visible no podra ser modificado.
