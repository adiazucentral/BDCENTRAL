Diagnóstico

Este maestro define el nombre de los  diagnóstico del sistema y de qué tipo son.

A continuación, se muestra una ventana típica con sus partes:paiu

      1.    Cuadro de texto de búsqueda.

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de los diagnósticos. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

 

      2.    Ordenamiento.

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente o por valor (falso o verdadero). El ícono que aparece en cada columna cambia según el orden.

 

3.    Cantidad. 

Este control muestra la cantidad de registros visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce un nuevo diagnóstico.

 

      4.    Fecha de última actualización - Usuario.

Aquí se muestra la fecha y el usuario que creó o modificaron el registro que ha sido seleccionado. Por defecto no aparece visible, solo cuando se selecciona una fila de la lista de los diagnósticos.

 

      5.    Botón ayuda. 

Aquí se despliega el manual de cada página.

 

 6.  Código.                                                                                                      

Es un campo alfa-numérico, acepta mayúsculas y minúsculas; el máximo de caracteres es de 10, es un dato que siempre será requerido y validado en caso de estar repetido.

 

      7.    Nombre.                                                                                                      

Es un campo alfa-numérico que identifica los diagnósticos, acepta mayúsculas y minúsculas ; el máximo de caracteres es de 60, es un dato que siempre será requerido.

 

8.    Tipo de diagnóstico.                                                                                                      

Define si el tipo del diagnóstico pertenece al ICD10, ICD9 o ninguno. Existe la posibilidad que esta lista pueda aumentar

 

9.    Estado.

Indica si el diagnóstico se encuentra activo o no. Al cambiar el estado muestra un cuadro de confirmación donde pregunta si se encuentra seguro de cambiar el estado o no. El control tiene forma de interruptor (switch).

 

      8.    Botón nuevo.

Este botón habilita los controles (excepto el interruptor de estado) para crear un nuevo diagnóstico. Si dichos controles tienen información, se limpian.

Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar, Imprimir e importar. Este botón por defecto debe estar habilitado.

 

      9.    Botón editar.

Este botón permite editar el diagnóstico seleccionado de la lista de datos y esta habilitado cuando se selecciona un registro de la lista. Se cargan los datos en la caja de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Imprimir e Importar.

 

      10.  Botón guardar.

Este botón permite guardar un diagnóstico  ya sea creado o modificado. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo, Imprimir e importar.

 

      11.  Botón cancelar.

Este botón cancela, limpia y deshabilita los controles del formulario incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Imprimir e importar.

 

      12.  Botón reporte

Este botón abre una ventana donde se muestra una previsualizacion del reporte, en este se podra escoger en que formato se desea exportar el listado de diagnósticos, este informe debe ser acorde al filtro y ordenamiento escogido en la tabla de datos del formulario inicial.

 

13.   Botón importar

Este botón abre una ventana donde se podra escoger el archivo en formato csv que se desea importar, el contenido del archivo debe contener tres columnas con los titulos "code", "name" y "type/id" (39. ICD10, 40. ICD9, 41. Ninguno) , seguido de los datos de los diagnosticos que van a ser importados, los textos no pueden contener el caracter (') puesto que el sistema no podria leer el archivo. 

 

Ejemplo:

 

 

Después de que el sistema reciba el archivo, y todos los datos sean válidos el sistema recargara la lista de diagnósticos, en caso contrario el sistema mostrara un listado de los datos que tienen inconsistencias, el nombre no puede estar vacío y numero del diagnóstico debe corresponder a los del sistema.

 

Importante: Se debe tener en cuenta la configuración del separador de listas de la aplicación, puesto que esta debe coincidir con el del archivo .cvs. Que va a ser importado, en caso contrario el sistema no podrá leer el archivo. Si los textos contienen comas (,) o puntos (.) se recomienda configurar el separador como punto y coma (;).  

 

 

      14.  Listado de diagnósticos.

Es una lista de datos que contiene dos campos: Código, Nombre tipo de diagnóstico y Estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro. La fila inactiva se mostrará en rosado.
