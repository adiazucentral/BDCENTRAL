Vigencia

Permite crear las vigencias, la cuales agrupan los precios de los exámenes por tarifa. Estas vigencias tienen una fecha de inicio y una de finalización, adicionalmente se puede activar automáticamente.

A continuación, se muestra una ventana típica con sus partes:

      1. Cuadro de texto de búsqueda.

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de las vigencias. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

 

      2. Ordenamiento.

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente o por valor (falso o verdadero). El ícono que aparece en cada columna cambia según el orden.

 

      3. Cantidad. 

Este control muestra la cantidad de registros visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce una nueva vigencia.

 

      4. Fecha de última actualización - Usuario.

Aquí se muestra la fecha y el usuario que creó o modificaron el registro que ha sido seleccionado. Por defecto no aparece visible, solo cuando se selecciona una fila de la lista de las vigencias.

 

       5 .Botón de ayuda

Aquí se despliega el manual de cada página.

 

6. Nombre.

Indica el nombre con que se reconocerá la vigencia, es un campo alfanumerico con una longitud de 60 caracteres y es obligatorio.

    

7. Fecha inicial.

Indica la fecha en que se activará la vigencia, esta no podra ser igual o estar entre las fechas inicial y final de otra vigencia.

 

8. Fecha final .

Indica la fecha de finalización de la vigencia, la cual se empleara para generar alarmas, esta no podra ser igual o estar entre las fechas inicial y final de otra vigencia

 

9. Aplicar automáticamente  .

Indica si la vigencia se aplica automáticamente cuando se cumpla la fecha inicial a la media noche.

 

10. Vigencia activa.

Indica si la vigencia se encuentra activa o no, este campo no podra ser modificado en este formulario, su fin es solo informativo.

 

      11 . Botón nuevo .

Este botón habilita los controles (excepto el interruptor de estado) para crear una nueva vigencia. Si dichos controles tienen información, se limpian.Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar y Reporte. Este botón por defecto debe estar habilitado.

 

12 . Botón editar

Este botón permite editar la relacion seleccionada de la lista de datos y esta habilitado cuando se selecciona un registro de la lista. Se cargan los datos en la caja de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Reporte.

 

      13. Botón guardar .

Este botón permite guardar una vigencia ya sea creada o modificada. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo e  Reporte.

 

      14. Botón cancelar .

Este botón cancela, limpia y deshabilita los controles del formulario incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Reporte.

 

      15.  Botón reporte

Este botón abre una ventana donde se muestra una previsualizacion del reporte, en este se podra escoger en que formato se desea exportar el listado de hojas de trabajo, este informe debe ser acorde al filtro y ordenamiento escogido en la tabla de datos del formulario inicial.

 

      16.  Listado de vigencias .

Es una lista de datos que contiene los campos de: nombre, fecha inicial, fecha final y estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro.
