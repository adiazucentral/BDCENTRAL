Tarifa

Formulario que define el nombre de las tarifas, las cuales se asocian a las vigencias y con esto a la facturación.

A continuación, se muestra una ventana típica con sus partes:

           

      1. Cuadro de texto de búsqueda.

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de tarifas. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

 

      2. Ordenamiento.

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente o por valor (falso o verdadero). El ícono que aparece en cada columna cambia según el orden.
 

      3.  Cantidad.

Este control muestra la cantidad de tarifas visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce un nueva tarifa.

 

      4. Fecha de última actualización - Usuario.                                                                                    

Aquí se muestra la fecha y el usuario que modificó la tarifa seleccionada. Por defecto no aparece visible, solo cuando se selecciona una fila de la lista de tarifas.


      5.  Botón ayuda .

Aquí se despliega el manual de cada página.

 

      6. Código.                                                                                                      

Es un campo alfa-numérico que hace referencia a la identificación única de la tarifa, es un campo requerido y  lo máximo son 10 caracteres

 

      7. Teléfono.                                                                                                      

      Este campo indica número de contacto del pagador es configurable en configuración.

 

      8. Dirreción 1.                                                                                                      

Este campo indica la dirección del pagador, es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres.

 

      9.  Ciudad.   

      Este campo Indica la ciudad donde se reside el pagador, es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres.

 

      10.  Página web.

 Este campo Indica la dirección de la página del  pagador, es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres.

 

      11.   Tarifa.

Este campo Indica el nombre que identifica a la tarifa. Es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres, es requerido y único.

 

      12.  Correo Pagador.

Este campo Indica la dirección electronica para el envio de resultados. Es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres.

 

13.  Dirección 2.                                                                                                      

Este campo indica la dirección alternativa del pagador, es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres.

 

      14.  Estado o departamento.                                                                                                        

Este campo indica el departamento o estado en donde queda la ciudad o  municipio, es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres.

 

      15.  Código Postal.                                                                                                      

Este campo indica el número designado por la oficina postal para el envío de correspondencia, es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres.

 

      16.  Observaciones.        

Este campo indica las notas necesarias con información relevante, es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres.

 

      17.   Verificar pago.

Este campo indica  si se requiere verificar al momento de imprimir si el paciente particular ha pagado la totalidada de la factura al momento de imprimir el resultado, permitiendo o no hacerlo dependiendo del saldo

 

      18.    Ítem por defecto.

Este campo especifica la tarifa que se cargará automáticamente al momento de crear una orden.

 

      19.    Ver precio en ingreso de ordenes.

Es un campo que permite mostrar u ocultar la tarifa para ingreso de ordenes.

 

      20.  Estado.

Es un campo que permite activar o desactivar una tarifa . Al cambiar el estado muestra un cuadro de confirmación donde pregunta si se encuentra seguro de

cambiar el estado o no.

 

       21.   Facturación Estados Unidos

Es un panel donde se mostrara la facturación de USA sólo se muestran si esta activo el módulo de billing USA.

A continuación, se muestra una ventana típica con sus partes:

      21.1  Tipo de pagador.

Campo de selección que indica si el pagador:

        ·    Aseguradora del estado

        ·    Aseguradora privada

        ·    Cuenta privada.

        ·    Particular u otros.

21.2  Asignar todas las cuentas.

       Indica si la tarifa se  le asigna a todos los clientes.                                                                                                                                                                   

       21.3   Aplicar homebound

  Indica si se realizan toma de muestra en casa.

  21.4   Verificar relación diagnóstico

  Indica si aplica la revisión para los diagnosticos, con respecto a los exámenes ingresados a la orden.

       21.5   Aplicar Diagnóstico.

  Indica si la tarifa seleccionada en el ingreo de ordenes se muestran los diagnósticos asociados a los exámenes.   

  21.6    Aplicar Venipunture.

  Indica si si se cobra la toma de la muestra.    

      

22.    Botón nuevo.

Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear un nueva tarifa . Si dichas cajas de texto tienen datos, se limpian.

Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar e Imprimir. Este botón por defecto debe estar habilitado.

 

 23.    Botón editar.

Este botón permite editar la tarifa seleccionada de la lista de datos y es habilitado cuando se selecciona de la lista. Se cargan los datos en las cajas de texto y al pulsar el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Imprimir.

 

       24.    Botón guardar.

Este botón permite guardar una tarifa ya sea creada o modificada. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo e Imprimir. 

 

       25.    Botón cancelar.

Este botón cancela, devuelve y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botones Imprimir y Nuevo.

 

       26.    Botón reporte

Este botón abre una ventana donde se muestra el reporte de las tarifas que se encuentran en la lista de datos. Se muestra los mismos registros filtrados y las opciones para guardar ya sea en PDF o en Excel.

 

       27.    Listado de tarifas.

       Es una lista de datos que contiene 3 campos: código, tarifa y Estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro. Las filas inactivas se mostraran resaltadas en rosado.
