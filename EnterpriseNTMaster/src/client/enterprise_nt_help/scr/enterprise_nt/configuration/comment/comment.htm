Comentarios

Los comentarios codificados, son textos predefinidos por el usuario, con el fin de evitar digitar textos repetitivos y largos en el registro de los resultados una y otra vez, mejorando el tiempo del proceso
A continuación, se muestra una ventana típica con sus partes:

1. Cuadro de búsqueda.
Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de los comentarios. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

2. Ordenamiento.    
Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente o por valor (falso o verdadero). El icono cambia según el orden.

3. Cantidad.    
Este control muestra la cantidad de comentarios visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce un nuevo comentario.

4. Usuario- fecha de última actualización.
Aquí se muestra la fecha y el usuario que modificaron el comentario que ha sido  seleccionado. Por defecto no aparece visible, solo cuando se selecciona una fila de la lista de comentarios.

5. Botón ayuda.    
Este botón despliega el manual de cada página.

6. Código.
Código alfanumérico identifica el comentario, adicionalmente sirve para el llamado del mismo al implementarlo, permite un máximo de 15 caracteres es un dato que siempre es requerido y único.

7. Comentario.
Detalle del texto a emplear, el cual se muestra al momento de capturarlo en registro de resultados.

8. Aplica para.
Informa en que módulo se emplea este comentario:

  • Aplicación, solo aplica en el LIS
  • Analizador: solo aplica a las interfaces con analizadores.
  • Ambos, Se seleccionan  las dos opciones. 

9. Diagnóstico.
Como el texto comentario es complemento de un resultado, indica si este comentario pertenece a un resultado normal, anormal o maligno.

10. Estado.
Muestra si el registro está activo o no

11. Botón reporte.
Este botón le ofrece al usuario una pre-visualización del reporte que va a ser generado. Se muestran las diferentes opciones para generar el reporte, como lo son PDF, EXCEL, en otros formatos más o si se desea se envía directamente a una impresora.

12. Botón cancelar.
Este botón cancela, limpia y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Reporte.

13. Botón guardar.
Este botón permite guardar un comentario ya sea creado o modificada. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo y Reporte.

14. Botón editar.
Este botón permite editar el comentario selecciono de la lista de datos y es habilitado cuando se selecciona un comentario de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Reporte.

15. Botón nuevo.
Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear una nuevo comentario. Si dichas cajas de texto tienen datos, se limpian. Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar y Reporte. Este botón por defecto está habilitado.

16. Listado de comentarios.
Es una lista de datos que contiene cuatro campos: Código, Aplica para, diagnóstico y Estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro. Las filas inactivas se mostrarán en rosado.
