Diagnóstico por prueba

Este maestro permite asignar a los diagnósticos los posibles pruebas ,paquetes y perfiles a emplear en ingreso de ordenes

A continuación, se muestra una ventana típica con sus partes:

     

      1.  Ordenamiento.

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente. El ícono que aparece en cada columna cambia según el orden.

 

      2.  Cuadro de búsqueda.                                                                                          

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de pruebas o diagnósticos. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles. 

 

      3. Cantidad. 

Este control muestra la cantidad de datos visualizados en la lista. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda .

 

4.  Áreas.                                                                                                     

Listado de áreas para  filtrar la pruebas.

 

      5. Usuario- fecha de última actualización.                                                                                                    

Aquí se muestra la fecha y el usuario que modificaron la prueba que ha sido  seleccionado. Por defecto no aparece visible, solo cuando se selecciona una  fila de la lista.           

                                             

      6. Botón ayuda.

Aquí se despliega el manual de cada página.

 

7.   Lista de pruebas.

Es una lista de datos que contiene dos campos: código y prueba. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro.

     

      8.  Botón guardar.

Este botón permite guardar los diagnósticos asociados. Al pulsarlo se deshabilitan los botón  Cancelar y el propio botón, y se habilitan el  botón  Reporte.

 

      9.  Botón cancelar.

Este botón cancela, se devuelve al estado anterior antes de modificar el formulario. Deshabilita los botón Guardar, y habilita el botón Reporte.

 

      10. Botón Reporte.

Este botón imprime una lista de diagnósticos que se encuentran asociadas a la prueba que selecciono.

 

      11. Lista de diagnóstico.

      Es una lista de datos que contiene dos  campos: Código, Diagnóstico y asignado. Dicha lista puede ser ordenada y/o filtrada por cada campo. el campo asignado se selecciona cuando se quieren asignar los Diagnósticos a la  prueba que se seleccionó en la lista de pruebas.
