Sedes

Una sede hace referencia a la distribución geográfica de los laboratorios, este valor permite agrupar las órdenes que se ingresan y se utiliza también en el esquema de seguridad del software.
A continuación, se muestra una ventana típica con sus partes:

1. Cuadro de búsqueda.
Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de sedes. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

2. Ordenamiento. 
Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente o por valor (falso o verdadero). El icono cambia según el orden.

3. Cantidad. 
Este control muestra la cantidad de sedes visualizadas en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce una nueva sede.

4. Usuario- fecha de última actualización.
Aquí se muestra la fecha y el usuario que modificaron la sede que ha sido  seleccionada. Por defecto no aparece visible, solo cuando se selecciona una fila de la lista de sedes.

5. Botón ayuda.    
Despliegue del manual de cada página.

6. Código.
Es un Identificador único de las sedes, es un campo numérico que permite un máximo de 2 caracteres es un dato que siempre es requerido y se valida si es repetido.
Nota: La sede primaria maneja el código 01, este código debe ser tenido en cuenta en la instalación inicial de la aplicación, no permite modificar ninguno de sus campos y no se visualiza en el maestro de Usuarios.

7. Nombre.
Es un campo alfa-numérico el cual almacena el nombre que identifica la sede. Acepta un máximo de 60 caracteres, este campo es requerido y único.

8. Abreviatura.
Es un campo alfa-numérico no mayor a 5 dígitos que identifica la sede en reportes, códigos de barras y listados; este campo es único.

9. Responsable.
Es un campo alfa-numérico que almacena el nombre de la persona responsable de la sede o el nombre del contacto.

10. Teléfono.
Es un campo alfa-numérico que almacena el número telefónico de la sede, acepta un máximo de 60 caracteres.

11. Dirección.
Es un campo alfa-numérico que almacena la dirección de la sede, acepta un máximo de 60 caracteres.

12. Correo.
Es un campo alfa-numérico que almacena el correo electrónico de la sede, acepta un máximo de 60 caracteres.

13. Rango mínimo.
Se almacena el número correspondiente al valor mínimo en donde inicia la secuencia de órdenes para la sede seleccionada.

14. Rango máximo.
Se almacena el valor máximo del rango de las órdenes a procesar para la sede seleccionada, debe ser mayor al rango mínimo.

Nota: Los campos de rango mínimo y rango máximo, cumplen con las siguientes condiciones:
·  Son requeridos y obligatorios cuando se maneja la configuración numeración de orden por sede, de lo contrario no se muestra, su valor por defecto es cero.
·  No se puede repetir con otra sede y una vez definido su valor no puede ser cero.
·  Para dejar el rango sin definir se deja en cero el mínimo y el máximo.
·  Se debe mostrar según la cantidad de dígitos definidos para la orden.

15. URL.
Este campo permite identificar la ubicación del backen cuando las sedes esta configurado como multisedes.

16. Estado.
Este campo permite activar o desactivar el registro seleccionado. Tiene forma de interruptor.

17. Botón nuevo.
Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear una nueva sede. Si dichas cajas de texto tienen datos, se limpian. Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar e Imprimir . Este botón por defecto está habilitado.

18. Botón editar.
Este botón permite editar la sede selecciona de la lista de datos y es habilitado cuando se selecciona una sede de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Imprimir .

19. Botón guardar.
Este botón permite guardar una sede ya sea creada o modificada. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo e Imprimir .

20. Botón cancelar.
Este botón cancela, limpia y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Imprimir .

21. Botón reporte.
Este botón imprime las sedes que se encuentran en la lista de datos. Se muestra un reporte con las mismas columnas y los mismos registros filtrados.

22. Listado de sedes.
Es una lista de datos que contiene cuatro campos: abreviatura, Nombre y Estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro; las filas inactivas se mostrarán en rosado.
