Tipos de orden

Maestro que define los tipos de orden que se manejaran el toda la aplicación. Ingreso de órdenes, listados, informes, estadísticas, consultas e Interfaces.

A continuación, se muestra una ventana típica con sus partes:

1.       1. Cuadro de búsqueda. 

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de tipo de orden. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

 

      2. Ordenamiento .  

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente. El ícono que aparece en cada columna cambia según el orden.

 

3. Cantidad .

Este control muestra la cantidad de tipo de órdenes visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce una nuevo tipo de orden.

 

      4.  Usuario- fecha de última actualización.

Aquí se muestra la fecha y el usuario que modificaron el tipo de orden que ha sido  seleccionado. Por defecto no aparece visible, solo cuando se selecciona una  fila de la lista de tipo de orden.

 

      5.  Botón ayuda.

Aquí se despliega el manual de cada página

 

6.  Código .

      Letra que distingue el llamado o selección del tipo de orden, el cual no se puede repetir, acepta mayúsculas y minúsculas, lo máximo es un carácter es un dato que siempre es requerido.

 

      7.  Nombre .

      Nombre del tipo de orden a definir el cual no se puede repetir Es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres, Es un campo requerido.

     

8.  Color .

      Color que se mostrará en ingreso de órdenes para distinguir cada tipo de orden.

 

      9.  Estado.

Este botón Indica si registro está activo o no, al modificar el estado siempre hay aparece un cuadro de dialogo que pide la confirmación.

 

      10.  Botón nuevo.

Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear un nuevo tipo de orden . Si dichas cajas de texto tienen datos, se limpian. Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar e Imprimir. Este botón por defecto debe estar habilitado.

 

      11. Botón editar.

      Este botón permite editar el tipo de orden seleccionado de la lista de datos y es habilitado cuando se selecciona un banco  de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario pero excepto 5 tipos de ordenes predefinidos, ( Tan pronto como sea posible, prioritario , re - llamado , Rutina y Urgencias )  los primeros tres solo  se   puede modificar el nombre , color y el estado, los últimos dos solo el nombre y el color . Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Imprimir.

 

      12.  Botón guardar.

Este botón permite guardar un tipo de orden  ya sea creada o modificada. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botones Nuevo e Imprimir.

 

      13.  Botón cancelar.

      Este botón cancela, limpia y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Imprimir.

 

      14. Botón reporte.

      Este botón imprime Los tipos de orden que se encuentran en la lista de datos. Se muestra un reporte con las mismas columnas y los mismos registros filtrados.

 

15. Listado de tipo de orden.

Es una lista de datos que contiene tres campos: Código, Nombre y Estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila     seleccionada siempre se mostrará resaltada en azul claro y las inactivas estarán resaltadas con rosado.

     

      Nota: Los tipos de órdenes predefinidos por el sistema estarán marcados con un ícono en forma de candado indicando que no se les podrá cambiar el código. Los registros de Rutina y Urgencia no se podrán desactivar.
