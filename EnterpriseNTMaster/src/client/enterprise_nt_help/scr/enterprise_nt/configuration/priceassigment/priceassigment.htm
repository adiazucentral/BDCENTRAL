Asignación de precios por tarifa

Este formulario permite crear y asignar los precios que se aplicarán a cada uno de los exámenes para el módulo de facturación y caja, agrupados por vigencias. adicionalmente permite exportar e importar los precios.

A continuación se muestra la ventana especial para la asignación de precios de pruebas por tarifa:

 

                                                                                                     [Imagen 01 - Asignación de precios por tarifa]

1. Ordenamiento.

Se usa el ordenamiento por columnas de menor a mayor o viceversa, ambas listas se organizan ya sea alfabéticamente o por valores numéricos. El ícono que aparece en cada columna cambia según el orden.

 

2. Cuadros de texto de búsqueda.
Se escribe o se introduce un valor alfa-numérico para filtrar elementos de las listas tanto de áreas como de pruebas. El filtro se da de manera dinámica; a medida que se vayan digitando caracteres, la lista va mostrando las filas coincidentes con las columnas visibles. 

3. Tarifa.
En esta lista desplegable se encuentran todas las tarifas activas para seleccionar una. 

4. Vigencia.
En esta lista desplegable se encuentran todas las vigencias para seleccionar una. 

5. Cantidad.

Estos controles muestran la cantidad de registros que están en vigencias, tarifas, áreas y pruebas.  Para los casos de las áreas y pruebas su valor cambia a medida que se vayan filtrando datos en cada cuadro de búsqueda.

 

6. Fecha de última actualización - Usuario.                                                                                                        

Aquí se muestra la fecha y el usuario que modificó los precios correspondientes al área seleccionada. Por defecto no aparece visible, solo cuando se selecciona una fila de la lista de áreas.

 

7. Botón ayuda. 

Aquí se despliega el manual de cada página.

 

8. Importar precios. 
Se pulsa en el botón con ícono y  tooltip de importación. Aparece la siguiente ventana:

 

 

En el botón con ícono de carpeta se abre un cuadro de dialogo para explorar el archivo con formato csv dispuesto a Abrir. Al seleccionar dicho archivo aparece el nombre en la caja de texto Archivo para importar. Luego aparecen las opciones para seleccionar el tipo de código con el que se evaluarán los exámenes a guardar. Si se selecciona Código LIS, el sistema valida las pruebas o exámenes por medio del código de los mismos, si se selecciona Código central, el sistema valida las pruebas que están en el archivo que concuerden con las pruebas que tienen código central asociadas al sistema central seleccionado. Luego se pulsa el botón importar y se guardan las pruebas con sus respectivos precios.

9. Actualización de precios mediante fórmula. 
Se pulsa en el botón con ícono y  tooltip de fórmula. Aparece la siguiente ventana: 

En esta ventana se selecciona la tarifa destino, es decir, la tarifa a la que se calcularán las operaciones matemáticas por medio de un valor con base en los precios de la tarifa origen seleccionado al principio del formulario. Si se seleccionan los signos de multiplicación y división, éstos siempre podrán calcular con valores decimales indistintamente de que la configuración para centavos esté deshabilitada. Si se selecciona el signo de división, éste tendrá como divisor un valor mínimo de 0.01. Aparecen las opciones de Redondear y Truncar, al seleccionar la primera, pues los precios se redondean al valor decimal más próximo, al no tener configurado los centavos, el redondeo se hace al entero en caso de calcular con multiplicación o división, si se selecciona la opción de truncar, los precios se les corta el valor dependiendo de la configuración de los centavos. Aparece la opción para cálculo Total o Parcial, el total calcula todas las pruebas de la vigencia y tarifa seleccionada, si se selecciona parcial, solo se actualizan los precios de aquellas pruebas que tengan valor 0 o no tengan precios asignados. Al pulsar el botón guardar, al fondo se observa la actualización de los precios en la grilla de pruebas.

10. Aplicar vigencia. 
Se pulsa el botón con tooltip de aplicar vigencia y se abre un cuadro de confirmación preguntando si se desea aplicar la vigencia seleccionada, este botón solo se habilita cuando la caja de texto de precio detecta algún cambio.

11. Exportar precios. 
Se pulsa el botón con ícono y tooltip de exportación y se genera un archivo con el nombre y formato PriceTest.csv, cuanta con 3 columnas: Code, Price y Abbr .

12. Caja de texto del precio.
Aquí se digita el precio que se habilita al momento de seleccionar una prueba, éste se digita con formato de moneda configurable por el sistema y al momento de perder el foco se guarda el precio para la prueba seleccionada. 

13. Listado de pruebas a asignar precios.
Es la lista de las pruebas que cuenta con las columnas de código, nombre y el precio que se habilita para ser digitado para cada prueba.  Se puede filtrar y ordenar, el precio aparece con formato de moneda configurable en el sistema. 

14. Botón Cancelar.
Se cancela la selección del área y se limpia la lista de pruebas con sus respectivas columnas.

15.  Botón Reporte.
Se genera un reporte de la lista de las pruebas con sus precios agrupadas por el área seleccionada. El reporte se puede exportar en formatos Excel o PDF.

16. Listado de las áreas activas.
Es la lista de las áreas activas que se visualizan con los campos de Código y Nombre de área. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro y en ese momento aparecerán las pruebas asociadas al área seleccionada, dichas pruebas se visualizarán en una lista con el campo de precio disponible para editar.
