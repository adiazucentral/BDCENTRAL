Homologación de pruebas

Este formulario hace una relación entre las pruebas creadas en el sistema y un código creado por el cliente en sus sistemas típicos de software de hospitalización para el manejo de la transmisión de las interfaces.

se pueden crear más de un código por examen, ya que se agrupan por el maestro de sistemas centrales.

 A continuación, se muestra una ventana típica con sus partes:

      

      1.  Cuadro de búsqueda.                                                                                          

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de sistemas centrales. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

 

2.   Ordenamiento.

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente. El ícono que aparece en cada columna cambia según el orden.

 

      3.  Cantidad.  

Este control muestra la cantidad de datos visualizados en la lista. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda .

 

      4.  Usuario- fecha de última actualización.                                                                                                    

Aquí se muestra la fecha y el usuario que modificó los códigos de homologación de las pruebas del sistema central selccionado . Por defecto no aparece visible, solo cuando se selecciona una  fila de la lista. 

 

      5.  Botón ayuda.

Aquí se despliega el manual de cada página.

 

      6.   Lista de áreas.

Es la lista de los sistemas centrales activos en el sistema y cuante con los campos de nombre del sistema central, reptir código y CUPS . Dicha lista puede ser ordenada y/o filtrada por cada campor. la fila seleccionada siempre se mostrará resaltada en azul claro y ene ese momento aparerám las pruebas asociadas al sistema central seleccionado, dichas pruebas se vizualizarán en una lista con sus respectivos códigos centrales, ubicados a la derecha.

 

7.    Cuadro de texto de búsqueda de pruebas.

Este botón imprime una lista de usuarios con su código central que se encuentran en el  sistema central que selecciono.

 

8.   Listado de pruebas y códigos centrales.

Es donde se visualizan las pruebas con sus respectivos código centrales . Se digita o se quitan los códigos centrales y al perder el foco se guarda.
