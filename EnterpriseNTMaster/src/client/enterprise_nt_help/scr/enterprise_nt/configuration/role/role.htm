Roles

Los roles definen el tipo de nivel seguridad de la aplicación, para posteriormente definir los accesos y acciones que se puede realizar en cada formulario de la aplicación.
A continuación, se muestra una ventana típica con sus partes:
 

1. Cuadro de búsqueda .  

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de roles. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

2. Ordenamiento.
Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente. El ícono que aparece en cada columna cambia según el orden.

3. Cantidad.
Este control muestra la cantidad de roles visualizados en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda o cuando se introduce un nuevo rol.    

4. Permisos para laboratorio.
 Seccion para asignar los permisos a los modulos de laboratorio y sus respectivas paginas. 

5. Usuario- fecha de última actualización.                                                                                                     
Aquí se muestra la fecha y el usuario que modificaron el rol que ha sido  seleccionado. Por defecto no aparece visible, solo cuando se selecciona una  fila de la lista de roles.

6. Botón ayuda. 
Aquí se despliega el manual de cada página.

7. Permisos para configuración.
 Seccion para asignar los permisos a los modulos de configuración y sus respectivas paginas

8. Rol.  
Identificador único del rol el cual no se puede repetir, De preferencia nemotécnico. Es un campo alfa-numérico que acepta mayúsculas y minúsculas, lo máximo son 60 caracteres, Es un campo requerido.

9. Administrador.
Campo lógico que indica si este tipo de nivel va a ser administrador o no.

10. Estado.
Indica si el rol se encuentra activo o no. Al cambiar el estado siempre pide una validación si se encuentra seguro de cambiar el estado.

11. Botón nuevo.
Este botón habilita las cajas de texto (excepto el interruptor de estado) para crear un nuevo rol. Si dichas cajas de texto tienen datos, se limpian. Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones Editar y Reporte . Este botón por defecto debe estar habilitado.

12. Botón editar.
Este botón permite editar los roles seleccionados de la lista de datos y es habilitado cuando se selecciona un rol de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Reporte.

13. Botón guardar.
Este botón permite guardar un rol ya sea creada o modificada. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilitan los botón Nuevo y reporte .

14. Botón cancelar.
Este botón cancela, limpia y deshabilita las cajas de texto incluido el interruptor del estado. Deshabilita los botones Editar y Guardar, y habilita el botón Reporte.

15. Botón reporte.
Este botón imprime los roles que se encuentran en la lista de datos. Se muestra un reporte con las mismas columnas y los mismos registros filtrados.

16. Lista de roles.
Es una lista de datos que contiene tres campos: Rol, Administrador y Estado. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro y las muestras inactivas estarán resaltadas con rosado.
