Impuesto por prueba

Formulario para configurar el impuesto por cada prueba.

A continuación, se muestra una ventana típica con sus partes:

 

 

1.     Ordenamiento.

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente. El ícono que aparece en cada columna cambia según el orden.

 

2.     Cuadro de búsqueda áreas.                                                                                           

      Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de las áreas. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles. 

 

3.     Cantidad.

Este control muestra la cantidad de áreas activas en el sistema. Su valor cambia a medida que se vayan filtrando valores en el cuadro de búsqueda.

 

4.     Cuadro de búsqueda pruebas.                                                                                          

      Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de datos de las pruebas. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

 

5.     Usuario- fecha de última actualización.                                                                                                     

Aquí se muestra la fecha y el usuario que modificaron el área que ha sido  seleccionada. Por defecto no aparece visible, solo cuando se selecciona una  fila de la lista de áreas.    

                                  

6.     Botón ayuda.

Aquí se despliega el manual de cada página.

 

7.    Impuesto.

Campo para configurar el el impuesto de cada prueba, el valor minimo de este campo sera 0 y el maximo sera 100, si no se configura ningun impuesto y el campo queda vacio,  el sistema a la hora de cargar valores tomara el de la configuración general  

 

8.    Botón guardar.

Este botón permite guardar los impuestos configurados a cada prueba.

 

9.    Botón cancelar.

Este botón cancela, todos los cambios no deseados que se hayan hecho en el formulario

 

10.  Botón Reporte.                                                                                                                                                                                                                         Este botón imprime un listado de las pruebas y el impuesto configurado para cada prueba, todos los cambios no deseados que se hayan hecho en el formulario,  Se muestra un reporte con las mismas columnas y los mismos registros filtrados.

 

11.  Lista de áreas.

Es una lista de datos que contiene cuatro campos: Código y área. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará resaltada en azul claro, el principal objetivo de esta lista es hacer más fácil la búsqueda de pruebas y su configuración.
