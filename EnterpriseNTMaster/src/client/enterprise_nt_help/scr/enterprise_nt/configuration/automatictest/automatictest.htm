Prueba automática

El maestro de pruebas automáticas define una prueba a adicionar en registro de resultados dependiendo el resultado de otro

A continuación, se muestra una ventana típica con sus partes:

 

1. Ordenamiento.

Se usa el ordenamiento por columnas de menor a mayor o viceversa, la lista se organiza ya sea alfabéticamente, numéricamente. El ícono que aparece en cada columna cambia según el orden.

 

 

2. Cuadro de búsqueda.                                                                                          

Se escribe o se introduce un valor alfa-numérico para filtrar elementos de la lista de pruebas. El filtro se da de manera dinámica; a medida que se vaya digitando un valor, la lista va mostrando las filas coincidentes con las columnas visibles.

 

 

3. Área. 

Área por la que desea filtrar la prueba. 

 

4. Cantidad.

Este control muestra la cantidad de pruebas visualizadas en la lista de datos. Su valor cambia a medida que se vayan filtrando valores en el cuadro de  búsqueda.

 

      5. Usuario - Fecha de última actualización.                                                                                                          

Aquí se muestra la fecha y el usuario que modificaron el valor de la prueba automática  que ha sido s aleccionada. Por defecto no aparece visible, solo cuando se selecciona una fila de la lista de pruebas.

 

6. Botón ayuda.

Aquí se despliega el manual de cada página.

 

 

7. Botón de eliminación.                                                                                                                                                                                                                   Este botón elimina la comparación que esta asociada con el examen seleccionado.

 

 

8. Signo.  

Representa al signo que debe tener el resultado para comparar el primer resultado. pueden ser =, >=, <= , <, >, <>, o entre .

 

9. Valor.  

Valor del resultado a validar acompañado con el signo. para los casos en que se use el signo entre, se debe capturar un resultado mínimo y un máximo. la longitud, depende de la configuración de cada prueba 

 

 

10. Prueba.  

Lista de prueba a adicionar en registro de resultados cuando se cumpla la condición.

 

 

 

11.  Botón nuevo.

Este botón habilita las cajas de texto para crear una prueba automática si dichas cajas de texto tienen datos, se limpian. Al pulsarlo, se habilita el botón Cancelar y se deshabilitan los botones  Editar y Reporte.

 

      12.  Botón editar.                                                                                                                                                                                                                             Este botón permite editar las pruebas automáticas de la lista de datos y es habilitado cuando se selecciona una prueba de la lista. Se cargan los datos en las cajas de texto y al pulsarlo el botón Editar, se habilitan todos los controles del formulario. Se habilitan los botones Guardar y Cancelar, y se deshabilita el botón Reporte.

 

      13. Botón guardar.

Este botón permite guardar una prueba automática. Al pulsarlo se deshabilitan los botones Editar, Cancelar y el propio botón, y se habilita el botón  Reporte.

 

 

14. Botón cancelar.

Este botón cancela, limpia y deshabilita las cajas de texto.  Deshabilita los botones Editar y Guardar, y habilita el botón Reporte.

 

 

15. Botón Reporte.

Este botón imprime las pruebas automáticas asociadas a la prueba que se encuentra en la lista de datos.

 

 

16. Lista de pruebas.

Lista de datos que contiene dos campos: Código, abreviatura y nombre de la prueba. Dicha lista puede ser ordenada y/o filtrada por cada campo. La fila seleccionada siempre se mostrará re saltada en azul claro.
