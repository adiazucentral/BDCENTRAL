Informes

El formulario de Informes de resultados es el encargado de imprimir toda la información del registro de resultados por cada orden con sus respectivas pruebas o exámenes. En el informe se visualizan datos principales del paciente como el nombre y número de historia o documento, datos básicos de la orden como la fecha de ingreso de la orden, fecha de impresión y número de la orden.

Se observan datos básicos de los resultados y otros complementarios que a continuación se entrarán a detallar.

Antes de generar la impresión o el archivo del reporte se deben tener en cuenta los siguientes aspectos:

Pre-requisitos.
  • Tener pruebas, médicos y pacientes configurados en el sistema.
  • Tener órdenes creadas con sus respectivas pruebas.
  • Que las muestras de las pruebas estén verificadas.
  • Que las pruebas tengan algún resultado.
  • Que los resultados de las pruebas estén validados.
  • Para imprimir el informe directamente se debe instalar el cliente de impresión como se muestra en la siguiente imagen.


A continuación se muestra la ventana en donde se generará el informe de resultados con sus respectivos filtros y variantes según los datos almacenados y llaves de configuración.

1. Rango de órdenes. En este bloque se filtran las órdenes por un rango de días no mayor a 30. Se selecciona un año y al momento de digitar en cada cuadro numérico, las órdenes se auto-completan con el mes, día y el consecutivo de n dígitos configurados en el maestro de Instalación.

2. Tipo de orden. Se selecciona el tipo de orden para filtrar, por defecto el valor que aparece es -- Todos --.

3. Destino de información. En este bloque aparecen tres opciones para generar la información requerida por el usuario. Dependiendo de cada una el formulario muestra u oculta otros filtros y los botones se van habilitando.

  • Impreso: Es la opción seleccionada por defecto, permite imprimir el informe de resultados directamente en una impresora instalada en el equipo cliente, acá se pueden generar copias. El primer botón en violeta se habilita.  Al pulsar este botón se van generando los archivos en cola de impresión del equipo cliente. Si no está la impresora instalada, se muestra un mensaje advirtiendo el caso.Verifique la conexión con el cliente de impresión.
  • PDF: Es la opción que genera los archivos de PDF con la información filtrada de los resultados de las pruebas de cada orden. El nombre de los archivos siempre posee el número de orden correspondiente. El segundo botón en rojo se habilita. Al pulsar dicho botón se descarga un comprimido con los archivos PDF de cada orden. Acá no se pueden generar copias.
  • E-mail: Es la tercera opción que oculta el bloque de Ordenamiento de impresión por: para mostrar el bloque Envío de correo a. Se visualizan un par de variables seleccionables para enviar correo electrónico a los pacientes y/o médicos involucrados en los resultados de los exámenes de cada orden, esto depende de la llave de configuración en el maestro de Instalación. El tercer botón en color verde se habilita. Al pulsar este botón se comienza a procesar el envío de correos electrónicos a los diferentes destinos. Para los médicos se comprimen los archivos de las órdenes que ellos gestionaron, para los pacientes se envían los archivos de las órdenes de sus resultados sin comprimir.
4. Modo de impresión.
Es el bloque donde se encuentran tres opciones que indican como se van a generar las impresiones o los archivos de cada orden consultada.
  • Finales: Al seleccionar esta opción y al pulsar alguno de los tres botones el informe se titula Informe Final, el estado de las pruebas pasa de validadas a impresas, solo se imprimen aquellas que estén en estado validado, por lo que no se podrán imprimir o reportar mas de una vez.
  • Copias: Al seleccionar esta opción aparece un control numérico para incrementar el número de copias. Las copias solo se habilitan cuando se selecciona la opción Impreso. Las copias de las órdenes solo se elabora si previamente las pruebas están con estado de Impresas.
  • Reimpresión: Esta opción es muy parecida a la de las copias, con la diferencia que acá solo se imprime una sola y siempre estará habilitada para cualquier opción del destino de información. El título del informe aparecerá como Reimpresión.
5. Adjuntos.
Este control aparece cuando la llave de configuración (Imprimir adjuntos) en el maestro de Instalación está activa. Al seleccionarla brinda la opción de generar archivos PDF adjuntos o se adjuntan imágenes al reporte.

6. Ordenamiento de impresión por.

Acá aparecen tres opciones para establecer un ordenamiento de los datos.
  • Filtro de demográficos: Este control se marca para ordenar los datos cuando se filtran los demográficos. Es independiente de las oras dos opciones por lo que se puede establecer un orden combinado.
  • Órdenes: Se organizan los datos generados por el número de la orden.
  • Historia: Se organizan los datos por el número de documento o historia del paciente.

7. Filtrar por áreas o pruebas.
En este bloque se gestionan unos filtros opcionales que se pueden establecer seleccionado Áreas, Pruebas y perfiles o Pruebas confidenciales. El valor por defecto es General. Cada opción que se seleccione despliega unos controles para ir escogiendo uno o varios items con la acción de autocompletar.

8. Filtrar por demográficos.
Es el bloque encargado de filtrar los datos por los demográficos fijos y codificados. Entre los demográficos fijos están las sedes, servicios, médicos, raza, clientes y tipo de documentos entre otros.

9. Informe de órdenes impresas.
Este control se marca con el objetivo de desplegar una información final sobre la acción de impresión, PDF o email. Su contexto cambia dependiendo de las opciones del destino de información. A continuación se muestra una de las ventanas que se visualiza al finalizar una acción.


La ventana posee una información de las órdenes y pacientes, tiene un botón para generar un reporte que puede ser impreso.

10. Nombre de archivo PDF.
En este bloque aparecen un par de opciones que dependen de una llave de configuración en el maestro de Instalación (Generar nombre de PDF con historia clínica). Solo se habilita si las opciones de destino de información para PDF y E-mail son seleccionadas.

  • PDF con número de historia: Se genera el archivo con el número de orden más el número de historia o documento del paciente.
  • PDF con nombre de paciente: Se genera el archivo con el número de orden más el nombre y apellido del paciente.
