Consultas
Este módulo permite consultar todas las ordenes que se le han ingresado a un paciente y sus correspondientes pruebas mediante diferentes filtros.

Este modulo se divide en dos opciones:

1. Consulta normal
Esta opcion permite visualizar el listado de ordenes con las repectivas pruebas que le han sido asignadas a un paciente, junto con un detalle de los demograficos de la historia clinica del paciente, mediante la aplicación de diferentes filtros:
  • Area: Este filtro permite buscar las ordenes de los pacientes en un area especifica, por defecto el valor de este combo estara para que realice la busqueda en las diferentes areas del laboratorio.

 

  • Sede: Este filtro permite buscar las ordenes de los pacientes en una sede especifica, por defecto el valor de este combo estara para que realice la busqueda en las diferentes sedes del laboratorio.

     

 

  • Filtrar por: Este filtro me permite realizar diferentes tipos de busquedas de las ordenes de un paciente:
    • Historia: en este filtro se habilitaran controles para que realiza la busqueda por el numero exacto de la historia del paciente, en caso de que tenga configurado el manejo del tipo de documento, se mostrara un control para la seleccion del mismo, para que el sistema realice la busqueda debera presionar la tecla enter sobre el campo de historia.

    • Apellido(s): en este filtro se habilitaran controles para realizar la busqueda por los apellido(s) exactos y genero del paciente,  para que el sistema realice la busqueda debera presionar la tecla enter sobre el campo de apellido(s).

 

 

    • Consulta del día: Al seleccionar esta opción se cargaran automaticamente las ordenes creadas en la fecha actual.  
    • Número de orden: en este filtro se habilitaran controles para realizar la busqueda por año exacto y consecutivo de la orden,  para que el sistema realice la busqueda debera presionar la tecla enter sobre el campo de orden.

    • Demográficos: en este filtro se habilitara una lista de selección para demográficos codificos o no codificados, en el caso de el demográfico que se seleccione sea codificado, el sistema cargara una lista para la selección del item demográfico, en caso contrario debera digitar el valor exacto del demográfico no codificado. 

Una vez aplicado uno de los filtros anteriormente mencionados se cargara la información de las ordenes de la siguiente manera: 

1.1 Listado de ordenes agrupadas por paciente.
1.2 Detalle de los demográficos de la historia.        
1.3 Columna para visualizar el código de cada prueba de la orden.       
1.4 Columna para visualizar el nombre de cada prueba de la orden.       
1.5 Columna para visualizar el resultado de cada prueba de la orden, si a la prueba no se le a registrado ningun resultado, saldra el texto segun configuración(Comentario para resultados pendientes).
1.6 Columna para visualizar los valores de referencia de cada prueba.        
1.7 Columna para visualizar el resultado anterior de cada prueba.       
1.8 Columna para visualizar la unidad de cada prueba. 
1.9 Agrupacion de las pruebas por area.      
1.10 Estados de la prueba, de derecha a izquierda su representación es la siguiente:

    

  • Columna para visualizar si a la prueba se le agrego un comentario.

  • Columna para visualizar si a la prueba se le agrego un antibiograma.

  • Columna para visualizar si a la prueba se le agrego un adjunto.      

  • Columna para visualizar si la prueba ya fue verificada.      

  • Columna para visualizar si la prueba ya fue impresa.      
    • 


1.11 Boton para imprimir el informe final de las pruebas cuando sus resultados han sido validados y no impresos       
2. Historia por prueba.
Esta opcion permite consultar una historia clinica especifica asociada a una prueba en particular, mediante la aplicación de diferentes filtros

2.1 Campo para la seleccion del tipo de documento, este solo sera visible si se tiene configurado el manejo del mismo.
2.2 Numero de historia del paciente por el que desea realizar la busqueda
2.3 Campo para la selección de la prueba por lo que desea realizar la busqueda. 

Una vez aplicado el filtro anteriormente mencionado se cargara la información de las ordenes del paciente que tengan asociadas la prueba seleccionada de la siguiente manera: 

2.4 Detalle de los demográficos de la historia.
2.5 Listado de las ordenes que pertenecen al paciente y que tienen asignada la prueba seleccionada.
2.6 Columna para visualizar el resultado de la prueba, si a la prueba no se le a registrado ningun resultado, saldra el texto segun configuración(Comentario para resultados pendientes).
2.7 Columna para visualizar los valores de referencia de la prueba.  
2.8 Columna para visualizar el resultado anterior de cada prueba.
2.9 Columna para visualizar la unidad de cada prueba.
2.10 Estados de la prueba, de derecha a izquierda su representación es la siguiente:

  • Columna para visualizar si a la prueba se le agrego un comentario.
  • Columna para visualizar si a la prueba se le agrego un antibiograma.
  • Columna para visualizar si a la prueba se le agrego un adjunto.
  • Columna para visualizar si la prueba ya fue verificada.
  • Columna para visualizar si la prueba ya fue impresa.
2.11 Columna apara visualizar la fecha de ingreso de la prueba.
2.12 Columna para visualizar la fecha de validación de la prueba.
2.13 Botón para imprimir reporte con los ultimos 20 resultados numericos validados de los diferentes ordenes que ha tenido el paciente, solo se habilitara para las pruebas de tipo numerico.  
2.14 Botón para imprimir reporte con los ultimos 20 resultados validados de los diferentes ordenes que ha tenido el paciente.
